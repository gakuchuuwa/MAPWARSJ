**这是一份历史教材，不是娱乐游戏。**

- 所有据点坐标必须**真实可考**
- 所有势力名称必须有**历史出处**
- 所有旗号必须符合**真实历史身份**
- **绝对禁止**：虚构、胡编、为算法妥协地理位置、为完美数据捏造历史

如果某项数据找不到历史依据 → **直接不收录**，不要编造。
**宁可缺少 1 个据点，不可错放 1 个据点。**

## 二、设计公理（不可动摇）

| 公理 | 说明 |
|---|---|
| **1 势力 = 1 据点 = 1 旗号 = 1 军 = 1 色** | 项目核心机制，所有功能都建立在此之上（见下） |
| ↳ **一据点** | 每个 `factionId` 在 `cities_v2` 中**仅对应一座城**；`STARTING_CAPITALS` 唯一；无第二城、无「势力领土多分点开局」 |
| ↳ **一旗号** | 玩家只见旗号汉字 + 据点名；见 §三、§四 |
| ↳ **一军** | 开局/募兵：每据点同时最多 **1 支现役军团**（`RecruitmentSystem.cityHasActiveLegion`） |
| ↳ **一色** | 每局 `FactionManager.addFaction` 分配 **唯一 HSL** → 旗号染色、军团贴图、领土同色（§10.2）；`factions.ts` 不写 hex |
| **以据点为主** | **据点名本位**：先留名气大地，再从此地史地身份（民族/政权/家族/州郡）立项定旗（§2.2、§三） |
| 最小据点间距 ≥ 50 km | 任意两据点须 ≥ 50 km；过近优先 **删小留大**（见 §2.1）；必要时可 **小幅微调 1–2 km** 凑够 50 km，严禁胡编大挪移 |
| **新建据点间距窗口 50–500 km** | **仅约束新建/改坐标**：与最近已有据点 **>50 km 且 <500 km**（见 §2.1.1）；全图存量城对可 >500 km |
| 道路单段距离（非硬顶） | **汉地核心**驿道宜较短（酒泉↔瓜州 ~300 km 量级可作参考）；**边陲**（东北/草原/西域/中亚/青藏等）**允许更长单段**，渔猎/游牧/沙漠走廊本即远距节点，**不以 333 km 上限约束据点密度或删城** |
| **坐标优先不改** | lat/lng 锚定真实史地；间距冲突优先删点；两难时可 **小幅微调 1–2 km**，严禁大挪移；**据点名可改**（防重、换更知名旧称），见 §2.2 |

### 2.1 据点间距过近：删小留大，必要时小幅微调（基本准则）

**任意两座据点**（含 `panjun` 叛军点）实测地理距离 **< 50 km** 时：

| 做法 | 说明 |
|---|---|
| ✅ **删一个，留名气大的** | 以**据点名（历史地名）名气**为首要基准，删小留大（见 §2.2） |
| ⚠️ **小幅微调可接受** | 两难无法删留时，可 **±1–2 km** 微调来勉强凑够 50 km；**严禁**大幅挪移或胡编新坐标 |
| ❌ **禁止双点并存** | 不得「两个都留着」指望寻路/缩放能扛 |

**删留比较顺序**（同距冲突时自上而下比较，命中即留）：

1. **据点名史地知名度** — 按**大多数历史观认同**比较（开封 > 新郑；襄阳 > 樊城；长安 > 渭南；太昭 > 秀巴堡；开封 > 中牟）
2. `big_city`（§六大城白名单 20 座）
3. 已挂正规旗号势力之首都（`factionId !== 'panjun'`）
4. `medium_city` > `small_city` > `pass`（关隘/哨所）
5. 长期治所、国都/州郡治所 > 堡寨、渡口、纯地标
6. 仍难分 → **交项目主人裁定**；若需微调 1–2 km 凑间距，也须在裁定中说明

**名气拿不准时的辅助判定（2026-06-17 立，仍仅限古代旧称）**：

| 步骤 | 做法 |
|------|------|
| 1 | 只在 **1900 年前可考** 的旧称之间比较（近代/江户后城名、明治后县名 **不得** 参与比武） |
| 2 | 仍难分 → 查 **维基百科**（中文优先，可对照日文） |
| 3 | **有独立词条** 的旧称 > 无词条或仅在他条一笔带过 |
| 4 | 仍难分 → **字数更多的古代旧称** 视为名气更大 |
| 5 | 间距 \< 50 km → **删小留大**（上表 1–6 顺序），或 ±1–2 km 微调；**禁止**为凑间距改大地名、大幅挪坐标 |

**禁止**：令制国府坐标自动优先于战国名城/古都（例：**月山富田** > 意宇国府；**太宰府** > 福冈城）；禁止为道路保留 `cityId` 而改据点名。

**删点连带（强制）**：**没有据点，就删除道路。** 任意据点从 `cities_v2.ts` 删除时，**必须同步**删除 `VectorRoadData.ts` 中所有以该 `cityId` 为 `startConnection` / `endConnection` 的道路段；**禁止**留悬空道路。实现方式：FactionEditor **`/api/batch-delete`**（删城时自动清道路），或 VectorRoadEditor / 清理脚本；**不得**只删城不改道路网。

**示例**：中原腹地开封、新郑与中牟过密 → 删 **中牟**，留开封/新郑；工布首都 **太昭** 与 panjun **秀巴堡** < 50 km → 删秀巴堡，留太昭。

### 2.1.1 新建据点间距窗口 50–500 km（2026-06-18 立，AI 必跑）

**适用范围**：向 `cities_v2.ts` **新增** `city_*` 条目，或**改动**已有 `lat`/`lng`。  
**不适用**：只改 `name` / `factionId` / `note` 且坐标不动。

| 检查项 | 规则 | 不通过 |
|--------|------|--------|
| **下限（硬顶）** | 与**全图每一座**已有据点距离 **≥ 50 km** | 任一 <50 km → **禁止写入** |
| **上限（硬顶）** | 与**最近**已有据点距离 **≤ 500 km** | 最近邻 >500 km → **孤悬过远，禁止新建** |
| **卡线** | 最近邻 ∈ [50, 55) km | AI **不得擅自新建**；须项目主人裁定 |

**合法窗口**：最近邻距离 **∈ [50 km, 500 km]**，且与任意城均 ≥50 km。

**命令（写盘前必跑，exit 0 才允许写入）**：

```bash
npx tsx scratch/audit_city_spacing.mjs --probe <lat> <lng> <据点名>
npm run city:spacing   # 写入后再跑全图 ≥50 km
```

**禁止**：

- 未经主人同意新增据点（见 §零）
- 不跑 `--probe` 就提交新坐标
- 距最近城 **>500 km** 的「填空孤点」（边陲扩点须先找 **≤500 km** 内已有节点，或问主人）
- 用卡线 50–52 km 在密集区硬挤新点

**与 §2.1 关系**：§2.1 管**过近**（删小留大）；本节管**新建**时的**过近 + 过远**双边界。道路单段 333 km 等非硬顶**不**替代本节。

### 2.2.0 据点本位录入：势力·武将·精锐（2026-06-22 主人定，**录入唯一入口**）

> **铁律**：**一切以据点为基准**，不以人物为基准，不以既有势力表为基准。  
> **禁止**「武将是白起 → 势力该是秦国」；**禁止**「秦国 → 白起该在天水」式反推。  
> **每一座录入据点**独立一套：据点名 + 势力 + 武将 + 精锐；**互不因名将归属而串城**。

**录入顺序（不可跳步）**：

| 步 | 问什么 | 产出 |
|----|--------|------|
| **① 据点** | 此坐标在古代有哪些称呼？（1900 前可考） | `cities_v2`：`name` + `lat/lng`；取**知名度最大**旧称 |
| **② 势力** | 这一带按下列优先级，有何可考身份？ | `factions.ts` 全名 + 旗号；**势力名 ≠ 据点名**（严格不等） |
| **③ 武将** | 哪些将领在此**战斗、驻守**过？ | `FactionGenerals` + `GeneralSkills`；须与**该据点**史地相关 |
| **④ 精锐** | 哪些部队在此**战斗、驻守**过？ | `*ExpeditionLegions`；须与**该据点武将**相关；定 T0–T3 |

**② 势力命名优先级**（无则向下试，命中即停；须有史料，宁缺毋滥）：

1. **民族** — 发源地或极深厚渊源  
2. **政权** — 曾在此建政且为**首都/割据中心**（正式国号去后缀作旗号，见 §4.4）  
3. **州郡治所** — 郡/州治（旗号取郡州专名，见 §4.5、§4.7）  
4. **家族** — 名门大本营（旗号单姓，见 §4.6）  
5. **地名** — 战场、水源、驿站、军台、要塞等**可考**旧称  

**③ 武将**：只挂**与该据点**有战斗/驻守史实的将领；**不得**因他城、他势力出名而挪将。  
**④ 精锐 tier**（与 §12.3.1 一致）：

| 级 | 标准 |
|----|------|
| **T3** | 史书有载、确有其兵（番号或泛称均可） |
| **T2** | T3 + 史上有名，或打过胜仗但无专名 |
| **T1** | T3 + 正史明确胜仗 + 有知名度（缺一不行） |
| **T0** | T3 + 正史以少胜多经典进攻战 + 有名气 |

**运行时（占城后）**：见 §12.2.1 — 将/精跟**据点录入**；旗号跟**占城势力**。

**反例（禁止再犯）**：

### 2.2 据点名本位：一地多史、多势可源（基本准则）

**锚点 = 据点名 + 坐标。** 建档、删留、定旗一律**从大地名出发**，不是先立势力再凑据点。

**一地多史**：同一真实地理位置上，史籍可考的叠合身份**均可**成为独立势力的来源——不限一种，历史上在此或以此为中心的：

| 身份类型 | 定旗走 §4.1 优先级 | 可立项为势力 |
|---|---|---|
| 民族 | 1 | ✅ |
| 政权（国都） | 2 | ✅ |
| 郡治 | 3 | ✅ |
| 家族大本营 | 4 | ✅ |
| 州名 | 5 | ✅ |

每一种身份各按 §4.1 **单独定旗号、单独占一座城**（仍守 1 势力 = 1 据点 = 1 旗号）。**势力从史地身份长出，史料从「这名大地」流出**；禁止先有空 `factionId` 再反向污染据点名或坐标。

> **「多势可源」是立项来源，不是一城多旗。** 从一名大地可盘点多种史地身份、分别立项为多个势力（各占一城、各定一旗）；**不是**一座城同时挂多面旗。

**据点名与势力名的史地来源（基本准则 — 必读）**：

> **据点可以是历史上此地的所有称呼；势力可以是此处所有的政权、民族、部族、家族、郡治、州名、可考地名。**  
> 据点名与旗号**各从该坐标（`lat/lng`）的史地清单中独立选取**，**不必**同一时代、**不必**同一身份类型，**禁止**因「据点名是三国、旗号是东晋」之类理由互相否决。

| 对象 | 规则 |
|---|---|
| **据点名** | 锚点 = **坐标 + 名字**。同一 `lat/lng`，凡 **1900 年前** 史籍可考的**此地旧称**，**均可**作 `name`——濡须口 / 东关 / 运漕等同地多名，择**知名度更大**者挂牌（例：含山东关坐标 → **濡须口** 优于 东关）。 |
| **势力 / 旗号** | 同一坐标，凡史上可考的 **政权、民族/部族、家族、郡治、州名、地名** 等身份，各有 §4.1 可考依据者，**均可**各立项为独立势力（各占一城、各定一旗）。**不是**据点名定什么时代，旗号就必须跟什么时代。 |
| **运行时** | 仍守 **1 坐标同时只挂 1 面旗**；争同一坐标时按 §2.1 删小留大，失利方改挂**第二 / 第三**可考据点（≥ 50 km）。 |

**示例（据点名 ≠ 旗号身份，合法）**：

| 坐标 | 据点名 | 旗号 | 说明 |
|------|--------|------|------|
| 含山濡须口 | **濡须口** | **北府** | 据点名取三国魏吴咽喉；旗号取东晋北府兵（同地可源，互不绑死） |
| 百色田阳 | **田阳** | **壮** | 据点名取田阳旧称；旗号取壮族土官 |
| 临汾 | **平阳** | **尧** | 据点名取尧都旧称；旗号取尧 |

**定旗**（仅针对**已立项**的某一势力）：对该势力按 §4.1 从优先级 1 试到 5 定旗号；**不得**用无据后世称呼凑旗；**但不得**用据点名的时代去否定该坐标上另一可考身份立项。

**立项须有依据**：可以从此地建立势力，但**禁止胡乱建立**——须满足 §4.1 某级**可考史料**（及 §4.8 宁缺毋滥）；名将路过、无据关隘、1900 后地名等**不算**立项依据。

**据点不动（精确定义）**：

| 一般不碰 | 允许调整 |
|---|---|
| **坐标** lat/lng（最好别改） | **据点名** `name`：须为 1900 前可考旧称 |
| 不为势力、不为凑间距而挪点 | **旗号 ↔ 据点名防重**：**优先改旗号**；大地名/区域主标（奴儿干城等）**禁止**仅为防重改成衙门/小地名；见 §2.4 |
| 不把大地名挪到小堡寨坐标上 | 换用**知名度更大**的同地旧称（例：西夜旗 @ **叶城**） |

**据点名**：全局**不重复**（严格相等判重，见 §三）；近距删留、防重换称时，**优先用知名度更大的历史地名**。

**与坐标的唯一占有**（运行时仍 1 点 1 旗）：

| 规则 | 说明 |
|---|---|
| 名气大地占原坐标 | 据点名更大者留住该 lat/lng；失利方改挂**第二 / 第三**可考首都（≥ 50 km） |
| 坐标不动 | 不为小势力平移 lat/lng；不为小势力把长安、洛阳级坐标让给堡寨 |
| 旗号可多元、坐标须唯一 | 长安可源出唐、隋、汉等多势力，但城中**同时只挂一方**；其余改别都（例：大顺 → 渭南） |
| 防重 | 旗号 ↔ 据点名冲突 → **优先改旗号**；不得为防重牺牲大地名主标（§2.4）；可 ±1–2 km 微调，禁大挪 |

**示例**：

- **长安** — 多朝国都；唐占长安，大顺改渭南。
- **皖城** — 隋唐舒州治所 → 旗号 **舒**（州名级）。
- **太昭** — 工布地区核心地名；工布旗号 @ 太昭，近距小堡删留时留太昭。



### 2.3 势力搬迁后原据点安置（基本准则）

当势力从据点 A 搬迁到据点 B（或势力被删/换都）时，原据点 A **不得留空、不得保留已迁走势力的 factionId**。**严格按以下顺序**逐一排查，命中即停：

| 顺序 | 排查项 | 说明 |
|------|--------|------|
| **① 民族** | 此处是否有民族的起源地，或与该民族有特别深厚的渊源？ | 例：发源地、祖居地、核心活动区 |
| **② 政权** | 历史上此处是否建立过政权（国都/割据中心）？ | 排除已迁走势力；若该政权已被其他势力代表（如钟离→淮），则跳过 |
| **③ 家族** | 此处历史上是否有特别大的家族（士族门阀、地方豪强）？ | 须史籍可考；家族名如琅琊王氏→王、兰陵萧氏→萧（按 §4.1 家族类） |
| **④ 州郡治所** | 此处历史上是否是州/郡的治所？ | 取该州/郡为势力名（例：濠州治所→濠） |
| **⑤ 州名取字** | 此处历史上叫什么州？ | 取州名第一个字（例：濠州→濠）；若第一步不重名则直接用 |
| **⑥ panjun** | 以上全无 | 归 **panjun**（叛军/中立） |

**禁止**：搬迁后原据点 factionId 留为已迁走势力、留空、或随意塞一个无关势力。
**注意**：排查范围包含该坐标及周边可考区域（如凤阳一带含蚌埠、钟离故地等）。

### 2.4 史地安置与防审计瞎改（2026-06-20 立，必读）

> **主人红线**：最怕 **瞎安置**——为跑通审计、凑数量、消红线而改大地名、乱挂势力/精锐/武将，且**无史地相关性**。

**铁律**：**史料相关性 > 玩家地理直觉 > 审计零违规。** 审计报红但无合规史料解 → **停手问主人**，禁止自作聪明。

| 原则 | 说明 |
|------|------|
| **三相关** | 据点（坐标+旧称）、势力/旗号（§4.1 该地身份）、精锐/武将（§12 锚点）各自须有可考依据；**三者不必同代，但须同地可源或地缘合理** |
| **大地名不动** | 奴儿干城、长安、开封等**区域主标**禁止仅为防重改成庙街、衙门通称、低知名度小地名 |
| **防重顺序** | ① **改旗号**（例：奴儿干城 + 旗面 `都卫`）② 失利方第二/第三首都 ③ 同地换**更知名**旧称 ④ 主人裁定 |
| **宁缺毋滥** | 无城不挂势、无据不挂旗、无专名不收精锐、无合理解不挂将；**审计通过不是改数据的理由** |

**反面教材（禁止再犯）**：奴儿干城 + 旗号「奴儿干」触防重 → ❌ 据点名改「庙街」；✅ 据点名仍 **奴儿干城**，旗面改 **都卫**（奴儿干都司）。

细则见 `.cursor/rules/no-arbitrary-placement.mdc`。

## 三、玩家可见 vs 代码内部（极重要）

### 玩家眼里只能看到 2 样东西（平时）

1. **旗号** — 旗帜上的 1-2 个汉字（例：`汉`、`魏`、`德川`、`大清`）
2. **据点名** — 地图上的城名（例：`洛阳`、`长安`、`江户`）

**远征跟拍时多第 3 样**（见 **§十二**）：跟拍军团改名后的 **远征番号**（3–6 字专名，如 `丹阳兵`、`督标秦军`）。  
→ 旗号、据点名、番号 **必须是三个不同的专名**（番号与据点名可共享汉字，见 §12.1），详见 §十二「三者防重」——**写进数据前必查，禁止再犯**。

代码内部的 `factionId`（如 `'edo'`）、`factionName`（如 `'德川'`）、`cityId`（如 `'city_edo'`）**玩家完全看不见**。

### 势力名称 vs 旗号（必读，二者不可混）

| 字段 | 文件 | 规则 | 示例 |
|------|------|------|------|
| **旗号** | `SandboxDisplayNames.ts` / `CityAssetManager.ts` | 玩家旗面汉字，**1–2 字**；按 §4.1 定；**禁「州」「国」字**（§4.0） | `壮`、`祖`、`宣`、`宿` |
| **势力名称** | `factions.ts` 的 `name` | **禁止单字**；须为完整史地称呼（民族/家族/州郡/政权等） | `壮族`、`祖氏`、`宣州` |
| **据点名** | `cities_v2.ts` 的 `name` | 1900 前可考旧称，全局不重复 | `田阳`、`宁远`、`鸠兹` |

**禁止**：`factions.ts` 写 `name: '壮'`、`name: '祖'`、`name: '宣'` 等单字势力名——单字只出现在**旗号**，势力名用 **壮族 / 祖氏 / 宣州** 等全称。

**正确对照**：

| factionId | 势力名称（factions.ts） | 旗号（SANDBOX_DISPLAY_NAMES） | 据点名 |
|-----------|-------------------------|-------------------------------|--------|
| `zhuang_d` | **壮族** | **壮** | 田阳 |
| `zu_d` | **祖氏** | **祖** | 宁远 |
| `danyang` | **宣州** | **宣** | 鸠兹 |
| `xu` | **徐国** | **徐** | 下邳 |
| `suzhou_d` | **宿州** | **宿** | 符离 |

**旗号禁字（旗面专用，与势力全名无关）**：写入 `SandboxDisplayNames` / `CityAssetManager` 的旗号汉字 **不得含「州」「国」**——势力名可写「宿州」「徐国」（`factions.ts`），旗面只写 **`宿`**、**`徐`**。详见 §4.0「旗号禁字红线」。

### 工作流：以据点为主（极重要）

**只问据点，不问势力。** 改据点归属时若原势力失城 → 直接删势力（5 文件同步），不要问「XX 势力怎么处理」。
**坐标不动。** 除非用户明确说「请修改坐标」，否则 lat/lng 一律不改。只列出坐标不等于指示修改。

**先有点，再挂旗；据点名本位，一地多势可源（§2.2）。** 维护顺序永远是：

1. **据点** — 地图上的真实历史地名 + 可考坐标（`cities_v2` 里的 `name` + `lat/lng`）**已经存在**；删留、间距冲突时**以据点名名气大为保留基础**；
2. **史地身份清单** — 在该坐标/地名上，盘点民族、政权、家族、州郡等；**有 §4.1 可考依据者才可立项**为独立势力（多势可源 = 立项来源，不是一城多旗；争同一坐标时名气大者占点，其余第二/第三首都）；
3. **旗号** — 为**每一个**已立项势力，按其史地身份在 §4.1 五级优先级**向下试**，定 1 面旗（1–2 汉字）；
4. **代码绑定** — `factionId` / `StartingCapitals` 只是实现「1 点 1 旗」的索引，**不是**策划入口。

| ✅ 正确思路 | ❌ 错误思路 |
|---|---|
| 皖城这个点 → 隋唐舒州治所 → 挂旗 **舒** | 先有一个叫「皖」的势力 → 找座城绑上去 |
| 商邑这个点 → 商州治所 → 挂旗 **上洛**（避与「商邑」重字） | 先定商州势力 → 把长安/别的城改名叫商邑 |
| 悬瓠这个点 → 袁氏大本营 → 挂旗 **袁** | 先收录袁氏 → 随便找个城当首都 |

**禁止**：为凑齐 `factions.ts` 里的空势力而**挪据点坐标**、**随意改据点名**、或**把势力硬绑到不符合史实的城**。  
（**例外**：旗号防重、换更知名 1900 前旧称而改据点名 → 允许，见 §2.2「据点不动」表。）  
若某 `factionId` 找不到合格据点 → **删势力或留空**，不得反向污染据点。

**冲突时**：**坐标不动**（lat/lng 不改）；据点名非必要不改（防重、换更知名旧称除外）。争的是**谁能挂这面旗**——知名度低者让位，改挂该势力历史上**第二 / 第三**据点的旗（仍须可考、≥ 50 km），而不是把大城坐标挪给弱势力。

### 重复判定规则

| 字段 | 判重规则 |
|---|---|
| **旗号汉字** | 字符串**严格相等** = 同一势力 |
| **据点名汉字** | 字符串**严格相等** = 同一据点 |
| `factionName` / `factionId` / `cityId` | **不参与判重**（只是代码标识） |
| 坐标 lat/lng | **不参与判重**（同一地点可能古今多名） |

### 示例

| 情境 | 判定 |
|---|---|
| 旗号 `德川` vs 旗号 `德川` | ✅ 同一势力（即便代码内 factionName 不同） |
| 旗号 `德川` vs 旗号 `德川幕府` | ❌ 不同势力（玩家眼里也是两个不同的旗） |
| 据点 `酒泉` vs 据点 `酒泉城` | ❌ 不同据点（玩家眼里是两个不同地名） |
| 据点 `北京` vs 据点 `北京` | ✅ 同一据点 |

**FactionEditor 必须严格按此判重，不做"包含/被包含"的模糊匹配。**

## 四、势力名称定名规则

**以 §三「据点为主」+ §2.2「据点名本位」为前提**：先确定**名气大的据点名与坐标**；再盘点该地历史上曾有的民族、政权、家族、州郡等身份，**各有依据者均可成势**（各 1 城 1 旗）。  
对**每一个**已绑定据点的势力，按下列规则定旗号（玩家眼里的「势力名」= 旗号 1–2 汉字；`factionId` / `factionName` 不参与判重）。  
不是先定势力再配城；**史地身份决定旗号**，找不到可考依据 → 该点不挂旗（或 `panjun`），禁止胡编。

### 4.0 定名算法（逐步执行，不可跳步）

**遍历单位 = 据点**（`cities_v2` 每一个点），不是 `factions.ts` 势力表。

```
对每个据点：
  ① 从优先级 1 试到 5；命中某一级且有可考史料 → 立刻停止，不得再向下试
  ② 按该级格式提取 1–2 个汉字作为旗号
  ③ 去掉禁止后缀（族、人、国、朝、氏、家等，见下表）；并确认旗号**不含禁字「州」「国」**（见旗号禁字红线）
  ④ 全局检索：旗号汉字不得与已有任何旗号严格相等
  ⑤ 旗号 ↔ 据点名防重（见 §4.1、§2.4）；冲突 → **优先改旗号**；大地名主标禁止仅为防重改名
  ⑥ 写入 5 文件（FactionEditor 批量模式）
  ⑦ 1–5 皆无合格依据 → panjun 叛军旗（宁缺毋滥，禁止生造）
```

**历史红线**：一切定名须有真实史料支撑；无依据宁可 **panjun / 不收录**，绝不生搬硬套现代名称或 AI 凑字。

**后缀禁令**（旗号上**绝不**出现）：

| 禁止后缀 | 处理 | 例 |
|---|---|---|
| 族、人 | 去掉后缀，保留史载民族专名 | 苗族→`苗`，僰人→`僰` |
| 国、朝 | 去掉后缀；政权档另按 §4.4 正式国号 | 黎国→`黎`，大理国→`大理` |
| 氏、家 | **仅 4 级家族**去后缀 | 袁氏→`袁`；**月氏**→`月氏`（氏为史籍专名一部分，见下） |

**旗号禁字红线**（2026-06-16 立，**旗面 1–2 字专用**；`factions.ts` 势力全名不受此限）：

| 禁字 | 规则 | 错误 ❌ | 正确 ✅ |
|---|---|---|---|
| **州** | 旗号汉字中**不得出现「州」** | `徐州`、`宿州`、`庆州` | 庆州→`庆`，宿州→`宿`，徐州→`徐` 或另取史地专名 |
| **国** | 旗号汉字中**不得出现「国」** | `黎国`、`百济国` | 黎国→`黎`，百济→`百济`（§4.4 正式国号去「国」） |

- **势力名 vs 旗号**：军情/叙事里可写 **宿州**、**徐国**（`getFactionName()`）；玩家旗面只渲染 **`宿`**、**`徐`**。
- **审计**：新增/改旗号前，检索 `SandboxDisplayNames.ts` + `CityAssetManager.ts`，凡含「州」「国」即违规，须改旗或改取字，**不得**把行政区全称直接写入旗面。

**旗号完整专名 vs 单字简称**（2026-06-16 立，**与 §4.4「禁止截字自造」同级**；细则见 `.cursor/rules/flag-no-single-char-abbrev.mdc`）：

| 原则 | 说明 |
|---|---|
| **只剥后缀** | 仅去「国」「州」「族」「人」「氏」等后缀；专名本体 **1–2 字完整写入**，不得再截成单字 |
| **日本令制国** | 与现有 **尾张、甲斐、伊贺、越后** 一致：伊势国→**伊势**、远江国→**远江**、常陆国→**常陆**；❌ 远江→`远`、上总→`总`、安房→`房` |
| **郡名/国号/部族** | **九江、百济、月氏、北条** 等写完整专名；❌ 百济→`百`、九江→`九` |
| **中国区州名** | 新增优先 **完整 2 字旧称**（上洛、宣府）；单字旗（宿、庆）仅沿用已有 §4.7 先例，**禁止新造单字简称** |

**民族专名 vs 截字**（优先级 1 最易混，必读）：

| 情形 | 写法 | 说明 |
|---|---|---|
| 去泛后缀 | 苗族→`苗`，僰人→`僰` | 「族」「人」是后人附加，可剥 |
| 史籍整词专名 | `月氏` ✅，`月` ❌ | 史书固定写法，**不得**截成单字 |
| 部族专名 | `汪古` ✅，`净` ❌ | 汪古部与克烈、蔑儿乞同级 → 用部族专名，**不能**被 5 级州名「净州」覆盖 |
| 回鹘、敕勒、柯尔克孜 | 以史籍原貌 1–2 字为准 | 几字写几字，不截字、不加「族/人」 |

### 4.1 五级优先级（1 最高，5 最低）

| 优先级 | 类型 | 选用条件 | 旗号格式要求 |
|---|---|---|---|
| **1** | **民族** | 该据点是某民族的**发源地**，或在古代有**极其深厚的原生历史渊源** | 史载民族/部族专名 1–2 字；去「族/人」等泛后缀；**不得**截断史籍整词（`月氏`≠`月`） |
| **2** | **政权** | 该据点历史上曾是某**原生政权的首都**（须可考，非仅治所/行在路过） | **正式国号**（见 §4.4）；如「大理」不写「理」，「大清」不是「清」，「黎」不是「黎国」；**姓/氏不是国号**（苻→家族，秦→国号） |
| **3** | **郡名** | 该据点是古代某著名**郡的治所** | 提取郡名（如 `筑前` → 太宰府） |
| **4** | **家族** | 某名门望族在此**发迹**且当地具有极高历史声望的**大本营** | **单姓**（如袁氏只写 `袁`） |
| **5** | **州名** | 以上 1–4 **皆无**可考依据 | 取州名**去「州」字**为旗号（庆州→`庆`）；**禁止**旗号含「州」字（§4.0 禁字红线）；或取城/邑/关名前缀（彭城→`彭`）；见 §4.7 |

**选用顺序**：从 1 向下试，命中**最高一级**即定，**不得**为凑数跳级、降级混用，也不得用低优先级覆盖高优先级（例：净州塞有汪古部渊源 → 1 级 `汪古`，不能改挂 5 级 `净`）。

**全局**：
- 旗号汉字**全局不许重复**（异称如 `鲜卑` / `慕容` 视为不同）。
- **【据点名年代】** 须为 **公元 1900 年以前** 史籍见载或可考的历史地名。**禁止**使用 1900 年后新定、改易、恢复的行政区名或俗称作据点名（民国新县名、1949 后更名等）。清代及更早近世地名可用，须可考（如 **瑷珲**、**喀什**）。
- **【旗号 ↔ 据点名防重】** 仅禁止**严格相等**（`通海`/`通海`、`焉耆`/`焉耆`）。一字共享不算重复（如 `邳`+`下邳`、`代`+`代县`、`咸`+`咸兴`），无需改据点名。例：✅ **南郑**+`汉`、**代县**+`代`、**下邳**+`邳`。
- 据点名 = 历史地名；旗号 = 势力身份；二者通常不同（例：旗号 `月氏`，据点 `蓝氏城`）。

### 4.2 批量录入格式

```
民族：月氏，据点：蓝氏城，坐标：36.7000, 69.6800
政权：百济，据点：慰礼城，坐标：37.5600, 126.9700
家族：藤原，据点：平安京，坐标：35.0100, 135.7600
郡名：筑前，据点：太宰府，坐标：33.5100, 130.5200
州名：庆，据点：安化，坐标：36.1000, 107.6300
```

### 4.3 数据维护阶段（2026-06 起）

**遍历单位 = 据点**，不是 `factions.ts` 势力表。对每个已收录据点，按 §4.1 判定应挂哪一级旗号；再同步 5 文件。

**第一步（进行中）**：凡**据点**属 **优先级 1·民族** 渊源地 → 核对 `cities_v2` 上该点 `factionId` / 旗号是否正确。

- 若无合格据点承载某民族旗号：**可新增**据点（坐标须可考，≥ 50 km），或改挂该民族**第二 / 第三**核心据点（仍须可考）。

### 4.8 无旗号据点（panjun）— 宁缺毋滥

**原则**：**没有的可以不挂，千万不能乱挂。**

| 规则 | 说明 |
|---|---|
| **默认** | 无明确 1–5 级可考依据 → **`panjun` 叛军旗**，或暂不立势力 |
| **禁止** | 为凑满地图而挂旗：无史实依据、名将路过、1900 年后地名、纯地标硬套州郡 |
| **允许升格** | 仅当该据点**同时满足**：① §4.1 某级优先级 **②** 史实可考 **③** 据点名 1900 前且与旗号不触 §4.1 防重 **④** 项目主人或审计清单确认 |
| **关隘/山口/哨所** | 无明确 1–5 级依据 → 维持 panjun；**有依据**时可升格，旗号可取关名前缀（如潼→`潼`），**据点名须改**避重 |

审计：`scratch/city_flag_audit.json` 中 `panjunAudit`；**未列入升格清单的，保持 panjun**。

**定义**：史书、典籍中该政权自称或被称的**国号**本身，不是后人概括、不是地名、不是姓氏、不是 AI 凑的双字。

**写法**（1–2 个汉字，以史籍原貌为准）：

| 正确 | 错误（禁止） | 说明 |
|---|---|---|
| `大理` | `理`、`大理国` | 国号即「大理」 |
| `大清` | `清` | 后金改国号后的正式称法；`清` 另指庆州等，≠ 大清 |
| `大周` | `常`、`周` | 张士诚国号「大周」，不能用常州/人名凑字 |
| `大顺` | `顺` | 李自成国号 |
| `西秦` | `兰` | 薛举国号，非兰州地名 |
| `黎` | `黎国` | 春秋黎国号即单字「黎」 |
| `百济` | `后百`、`甄` | 后百济国号仍作「百济」；禁止截字自造 |

**绝对禁止**：

1. **自造简称 / 截字** — 如把「后百济」写成 `后百`，把「大周」写成 `常`。
2. **后缀「国」「族」「人」** — 如 `黎国`、`黎族`、`俚人`（民族另按 §4.1，不走政权国号）；旗号**不得含「国」字**（§4.0 禁字红线）。
3. **用地名、州名、姓代替国号** — 如西秦写 `兰`、大清与庆州 `庆` 混淆。
4. **为避撞旗瞎编** — 若正式国号与已有旗号重复（如两「百济」），须项目主人定夺（删并、分时期、第二首都），**不得**捏造新旗号。

**撞旗时**：正式国号重复 → 交项目主人裁定；**不能**用非国号字顶替。

### 4.5 郡名旗号（优先级 3 专用）

**定义**：该据点是古代某著名**郡的治所**（可考）；旗号 = **郡名**本身，不写「郡」字。

**写法**：

| 正确 | 错误（禁止） | 说明 |
|---|---|---|
| `九江` | `九`、`江` | 九江郡治柴桑/浔阳 → 旗号「九江」 |
| `筑前` | `筑` | 日本令制国名可考时同列 |
| `高唐` | `平原` | 郡治据点名「平原」时，旗号不得与据点名相同 |

**绝对禁止**：

1. **郡治所不符** — 郡内非治所不得挂郡名（主人明确接受的除外，如河内@获嘉）。
2. **旗号 = 据点名** — 如平原郡治据点「平原」时，旗号不能仍写「平原」。
3. **截字** — 如「陇西郡」写「陇」（陇西郡 ≠ 陇州，见 §4.7）。

审计清单见 `scratch/county_seat_audit.json`。

### 4.6 家族旗号（优先级 4 专用）

**定义**：该据点是某**名门望族发迹、且在当地具有极高历史声望的大本营**（可考）；旗号 = **单姓**。

**写法**：

| 正确 | 错误（禁止） | 说明 |
|---|---|---|
| `袁` | `陈郡袁`、`袁氏` | 陈郡袁氏 → 只写「袁」 |
| `王` | `琅琊王` | 琅琊王氏 → 只写「王」 |
| `欧阳` | `欧阳氏` | 复姓以史籍原貌为准（2 字） |

**绝对禁止**：

1. **非发迹大本营** — 仅成名、任官、路过、战场，不算家族据地。
2. **前缀/后缀** — `郡`、`氏`、`族`、`人` 等（如 `曹氏`、`陇西李`）。
3. **旗号 = 据点名** — 违反「1 势力 1 据点」语义。

审计清单见 `scratch/family_state_audit.json`（`priority: 4` 条目）。

### 4.7 州名旗号 + 防重（优先级 5 专用）

**定义**：优先级 1–4 皆不适用时，取该地古代最著名的**州名**，提取 1–2 字为旗号（**必须去「州」字**；旗号禁含「州」「国」，§4.0 禁字红线）。**日本令制国**不适用本节单字截（如 `远`），须写 **远江、常陆** 等完整二字专名（§4.0「旗号完整专名 vs 单字简称」）。

**写法**：

| 正确 | 错误（禁止） | 说明 |
|---|---|---|
| `宿` | `宿州` | 宿州治符离 → 旗号「宿」，势力全名仍写「宿州」 |
| `徐` | `徐州` | 徐州境 → 旗号「徐」等，**不得**写「徐州」 |
| `庆` | `庆州` | 庆州治所据点须叫 **安化** / **合水** 等旧称，**绝不能**叫「庆州」 |
| `上洛` | `商` | 商州治 **商邑** 时，旗号不能写「商」（与据点名重复），可用 **上洛** 等旧称 |
| `舒` | `皖` | 舒州治 **皖城** 时，旗号不能写「皖」（与据点名重复） |
| `宣` | `宣州` | 宣州辖鸠兹 → 旗号「宣」 |
| `漯` | `郾` | 郾城属漯河；旗号取 **漯**，据点仍叫郾城（「郾」≠旗号「漯」） |
| `宣府` | `宣` | 明宣府(宣化) 与 唐宣州(芜湖) 须分旗 |
| `代` | `代`+**代邑** | 旗号可取「代」；据点须改 **代县** 等旧称，不能叫「代邑」 |
| `潼` | `潼`+**潼关** | 旗号可取「潼」；据点须改关史旧称，不能仍叫「潼关」 |

**关隘据点（`type: pass` 或名含「关/隘/峡/渡/塞/谷」）**：

- **默认 panjun**（52 面叛军旗），无 1–5 级可考依据时不升格。
- 若升格：旗号可取关名前缀（潼关→`潼`），**据点名须改为** 1900 前可考的其他旧称，**不能**仍叫「潼关」且挂「潼」。
- **禁止**：无依据乱挂、1900 年后关名、据点名与旗号触 §4.1 防重而未改名。

**强力防重机制**（全局执行）：

1. **旗号汉字全局唯一** — 写入前检索 `SandboxDisplayNames.ts` + `CityAssetManager.ts`；已占用则**不得**硬塞。
2. **【防重红线】** — 旗号与据点名**严格相等**禁止。**XX关/城/邑/州** 且旗号 = **XX** 时：**优先改旗号**；**区域主名/大地名**（奴儿干城等）**禁止**仅为防重改成衙门/小地名（§2.4）。关隘可换同关其他可考旧称；仍无解 → 主人裁定。
3. **【年代红线】** — 据点名（含为避重而改的名）须 **1900 年以前** 可考；禁止 1900 后新定地名。
4. **撞旗处理顺序** — ① 改绑第二据点 ② 用更完整历史名称（如 `宣府` vs `宣`） ③ 改据点旧称 ④ 删并势力 ⑤ **禁止**自造非史实新字。
5. **州 ≠ 郡** — 陇州(汧源) 旗号「陇」≠ 陇西郡；不得混用优先级 3 规则。

审计清单见 `scratch/family_state_audit.json`（`priority: 5` 条目）。

### 6.1 中城标准（medium_city）

**必须同时满足以下两条**：

| 条件 | 说明 |
|------|------|
| **① 政权都城或州郡治所** | 历史上某政权（王朝/王国/割据）的正式都城，**或**州/郡/府的治所城市 |
| **② 人口 ≥ 10 万** | 历史峰值人口不少于 10 万（有城垣/街市/人口聚居） |

**小城条件**：以上任一不满足 → small_city

> 中城是区域性重要城市，大城是跨朝代全国性地标（门槛见 §六）。

## 八、地图缩放锁定 zoom=9（2026-05-30 立）

**任何操作都必须保持 `map.getZoom() === 9`，绝对禁止切换缩放。**

适用所有场景：
- 选道路 / 切换路径 / 移动据点 / 自动连完成 / 审计结果定位
- 编辑器面板任何按钮的副作用
- 报告/弹窗关闭后

实现方式：
- 用 `map.panTo(latLng, { animate: true })` 平移，**不要用** `map.fitBounds()`
- 若当前 zoom ≠ 9，用 `map.setView([lat, lng], 9)` 强制设回
- 代码常量统一引用：`VectorRoadEditor.LOCKED_ZOOM = 9`

**违反此规则 = 用户体验破坏**，必须 revert。

- 全部参见 `src/data/cities.ts` 顶部 1-5 条注释

### 10.1 势力色块（领土）— 默认关闭

### 10.2.1 旗号汉字黑白（2026-06-09 立）

玩家看见的旗号 **1–2 个汉字** 必须与旗面底色**对比分明**：深旗白字、浅旗黑字。

| 项 | 约定 |
|---|---|
| **唯一分界** | 旗面亮度 `lum`（ITU-R BT.601：`0.299R+0.587G+0.114B`，0–255）与 **`FLAG_TEXT_LUM_THRESHOLD = 128`** 比较 |
| **深旗**（`lum < 128`） | **白字** fill `#f0f0e8` + **黑边** stroke `rgba(0,0,0,0.80)` |
| **浅旗**（`lum ≥ 128`） | **黑字** fill `#1a1a1a` + **白边** stroke `rgba(255,255,255,0.70)` |
| **适用范围** | 固定色（`HistoricalFactionColors.ts`）与每局随机色（`FactionManager.getFactionColor`）**同一阈值**，禁止固定色另设 `<50` 等第二套 |
| **亮度来源** | 优先 `HistoricalFactionColors[factionId]`；否则 `FactionManager.getFactionColor(factionId)` |
| **强制白字例外** | `FLAG_TEXT_WHITE_STYLE_FACTIONS`：`yue_d` 岳（黑旗白字，戚继光《纪效新书》等可考） |
| **强制黑字例外** | `FLAG_TEXT_BLACK_STYLE_FACTIONS`：`han_d` 汉（赤旗黑字，史料可考） |
| **叛军 panjun** | 不渲染汉字（无字旗） |
| **单一真理** | 常量 `FLAG_TEXT_LUM_THRESHOLD` @ `HistoricalFactionColors.ts`；逻辑 `CityAssetManager.resolveFlagTextIsDark` |
| **审计** | `npm run flag-text:check` → 列出全部固定色势力 lum 与黑白判定；新增固定色须过审 |

**禁止**（史料例外除外）：

- 深旗（秦、苻、大辽、大明等 `lum < 128`）用黑字 — **除外** `FLAG_TEXT_BLACK_STYLE_FACTIONS`
- 浅旗（商、唐、大元等 `lum ≥ 128`）用白字 — **除外** `FLAG_TEXT_WHITE_STYLE_FACTIONS`
- 在 `resolveFlagTextIsDark` 外硬编码 fill/stroke 而不走 lum 判定

## 十二、远征精锐（2026-06-11 立，AI 必读）

> **项目主人多次重申**：精锐不是另占一城，也不是用人名立势力；玩家眼里旗号、据点名、番号 **三者不得完全相同**；**番号与据点名允许共享汉字**（例：武川镇 + 武川镇军），旗号仍须与据点、番号防重（§12.1）。  
> 本节为远征精锐的**唯一真理**；与 `docs/GAME_DIRECTION.md`「远征细则」冲突时，**本节 + 史料**为准。

### 12.1 玩家可见：三样东西，三个专名（极重要，重复三遍）

| # | 名称 | 玩家在哪看见 | 示例 |
|---|------|-------------|------|
| ① | **旗号** | 地图据点旗面 1–2 字 | `山越`、`潼津`、`唐` |
| ② | **据点名** | 地图城名标注 | `宛陵城`、`潼关`、`长安` |
| ③ | **远征番号** | 跟拍军团名（远征下令后改名） | `丹阳兵`、`督标秦军`、`玄甲军` |

**红线**：

1. **严格不相等** — 旗号 ≠ 据点名 ≠ 番号（任意两两**字符串不得完全相同**）。
2. **共享汉字** — 仅约束 **旗号** 与其余二者：
   - ❌ 旗号 ↔ 据点名：不得共享汉字（例：❌ 旗号 `潼津` + 据点 `潼关` 共享「潼」）。

   - ✅ **番号 ↔ 据点名：允许共享汉字**（主人 2026-06-17 定；例：✅ 据点 `武川镇` + 番号 `武川镇军`）。
3. **§4.1 据点防重仍适用** — 据点名呈 `XX关/城/邑/州` 且**旗号** = `XX` → **改据点名**，不改旗号（与番号↔据点共享无关）。

- **数据录入**：`*ExpeditionLegions.ts` 仍写 `factionId → 番号`；经 `STARTING_CAPITALS` 推导为 `cityId → 番号`（`CITY_ELITE_LEGIONS`）。

### 12.3.1 精锐 tier 判定标准（2026-06-17 立）

| 级别 | 判定标准 | 开战战力乘数 | 配额 |
|------|---------|-------------|------|

## 十三、战斗立绘分配（三级规则 — 2026-06-21 定，AI 必读）

> **设计定案（2026-06-21 主人裁定 + 评审通过）**：三级规则 **无根本缺陷**，作为战斗立绘 **唯一方案** 长期有效。AI **不得**擅自改优先级、恢复 `portraits/`/`avg/`、或因审计/404 删改 `FactionGenerals` 占位路径。  
> **主人原话定案**：立绘按 **文化 → 政权夹 → 专属人物** 三级分配。  
> **单一实现**：`src/config/portrait_defaults.ts`（池 + `resolvePortraitAssetPath`）；将领表 `FactionGenerals.ts`；战斗 UI `CombatUI.setPortrait`。

### 13.1 三级优先级（从上到下，命中即停）

| 级 | 对象 | 规则 | 代码 / 数据 |
|---|---|---|---|
| **① 文化区** | **每个据点势力**（无专属政权夹、或无将的普通军团/守军） | 势力首都 `cities_v2.region` → 14 文化区之一 → 从该区 **文化夹随机池** 抽一张 | `REGION_PORTRAIT_POOLS`、`getFactionCultureRegion` |
| **② 政权专属夹** | **有独立素材夹的政权**（如秦→`qin/`、唐→`litang/`、武周→`wuzhou/`） | 从该 **政权夹** 随机抽；映射在 `FACTION_PORTRAIT_POOLS` | `FACTION_PORTRAIT_POOLS[factionId]` |
| **③ 专属人物** | **FactionGenerals 里的名将** | **先** 磁盘上该将 **专图**（`portrait` 路径）→ **无图则** 该将所属 **政权夹随机** → **再无则** 该势力 **文化区夹随机** | `getFactionGeneral` → `resolvePortraitAssetPath(general.portrait, { factionId })` |

**② 与 ① 的关系**：势力若在 `FACTION_PORTRAIT_POOLS` 有映射，**优先用政权夹**（② 覆盖 ①）；未映射的势力才纯走文化区池（①）。

**③ 与 ②① 的关系**：名将 **永远先试专图文件名**（如 `jinling_tandaoji.png`）；专图未补齐时 **不得** 跨区乱抽，只在 **本政权夹 → 本文化夹** 内随机兜底。

### 13.2 素材目录约定

| 类型 | 路径 | 说明 |
|---|---|---|
| 文化区池 | `public/assets/zhongyuan/`、`shuguo/`、`riben/` …（14 区对应夹） | 一夹 = 一文化；同夹内 **禁止 MD5 相同 PNG**（`npm run portrait:folder-audit`） |
| 政权专属 | `public/assets/qin/`、`litang/`、`wuzhou/` … | 仅 **有独立包** 的政权；在 `FACTION_PORTRAIT_POOLS` 登记 |
| 名将专图 | 同上政权夹或文化夹内，文件名 `{factionId}_{generalId}.png` 等 | `FactionGenerals.portrait` = **预留路径**；文件不存在时走 13.1③ fallback，**不删表项** |
| **废弃** | `portraits/`、`avg/portraits/`、`REGION_FIELD_PORTRAIT` | **禁止** 恢复或新建引用 |

### 13.3 运行时谁用哪一级

| 场景 | 行为 |
|---|---|
| 普通军团创建 | `getRandomFactionPortrait`：② 政权池 → ① 文化池 |
| 守军 / 无 portraitPath 单位 | `getCombatPortraitPath`：同上 |
| ≥4 万挂将军团 | `LegionManager` 设 `army.portraitPath = getFactionGeneral(...).portrait`（已 resolve） |
| 战斗 UI 显示 | `CombatUI.setPortrait`：场次自选 → 军团固定 path → generalId → 文化随机 |

**抽签一次跟定**：军团 `portraitPath` 在创建时定死，同军同脸；守军可每场随机（无 path 时走池）。
