【Project Qin】秦汉战争地缘大推演·执行手册 (Part 1 核心篇)
[!IMPORTANT] 最高指导原则 本项目致力于制作学术级、GIS地理基准的战史复盘视频。 叙事核心：采用**“AVG文戏 + MAP武戏”双轨结构，配合“全知视角史诗旁白”**，拒绝低幼化的角色对话。 史观基准：文戏考据《中国军事通史》，武戏考据《历代战争史》。

第一部分：核心方针与方法论 (Core Methodology)
1. 双轨并行制 (The Dual-Track Policy)
本项目在内容构建上严格切割为两条平行的逻辑轨道，确保“考据”与“演出”的学术分离：

Track A - 视觉推演轨 (Mapping / 武戏)

核心依据：《历代战争史》（台湾三军大学编）。

执行标准：提取书中关于行军路线、地形关隘、兵力部署的数据，在 GIS 卫星高程地图上进行精确的兵棋推演。

功能：解决“怎么打”的问题 (Tactics & Execution)。

Track B - 内容骨架轨 (Narrative / 文戏)

核心依据：《中国军事通史》（中国人民革命军事博物馆编）。

执行标准：提取书中关于地缘政治、战略决策、朝堂博弈的论述，通过 AVG (Visual Novel) 形式展现决策瞬间。

功能：解决“为什么打”的问题 (Strategy & Motives)。

2. 制作模式：分段流水线 (Segmented Production)
为了保证工业化量产质量，每一集视频在物理上拆分为三个独立制作段落，最后在剪辑软件中合并：

Part 1：战略决策 (AVG) —— [静态] 宫廷背景 + 人物立绘 + 史诗旁白。

Part 2：战役执行 (MAP) —— [动态] 卫星地图 + 兵棋推演 + 战术箭头。

Part 3：战果结算 (UI) —— [数据] 结果展示 + 局势复盘面板。

3. 叙事风格：全知旁白 (Omniscient Voiceover)
绝对禁区：严禁使用角色第一人称对话（如：“李斯：大王，我们要打赵国”）。

执行标准：统一使用第三人称史诗解说（史官视角）。

范例：“公元前236年，依据《军事通史》记载，李斯向秦王剖析了赵国北线空虚的致命破绽...”

文风规范：现代古典风 + 硬核军事术语（如：战略真空、内线作战、钳形攻势、脊柱折断）。

4. 五级地理叙事级联 (The 5-Level Geospatial Cascade)
此模型对应军事科学中的“战略-战役-战术”三级结构，并细化为适合 GIS 的五层变焦逻辑：

Level 1 (Strategic): 地缘格局
镜头：Zoom 7 (中原战略全图)。
任务：【看大局】。展示战前七国态势，明确“谁强谁弱”。

Level 2 (Grand Tactical): 战略决策
镜头：Zoom 8 (战区/国境线)。
任务：【找窗口】。指出哪怕只存在一个月的“战略窗口期”。

Level 3 (Operational): 战役行动 —— *核心 Hook 放置处*
镜头：Zoom 9 (行军路线/地形)。
任务：【选路径】。展示常规路线 vs 奇袭路线的对比。

Level 4 (Tactical): 战斗执行
镜头：Zoom 10 (局部战场/城池)。**注：仅在史料支撑时使用。**
任务：【看细节】。具体的兵棋碰撞、特效演出。如果史料不足，用“兵棋撞击+特效”代替微观厮杀。

Level 5 (Outcome): 战果结算
镜头：Zoom 8-9 (回拉视角) / UI 面板。
任务：【算代价】。地图变色，数据展示。战果通常需要“拉远”看版图变化，而非推近。

第二部分：技术架构与工作流 (Technical Workflow)
本系统采用**“从数据到像素” (Data-to-Pixel)** 的四层金字塔架构，确保从史料挖掘到画面渲染的全链路自动化。

1. 情报层 (Intelligence Layer) —— [基石]
执行者：NotebookLM

核心职责：ETL (数据抽取与清洗)。作为项目启动的绝对基石，负责对核心史料进行分段爆破。

执行标准：

文戏数据源：针对《中国军事通史》，提取“原话引用”、“人物性格”、“决策动机”。

武戏数据源：针对《历代战争史》，提取“兵力数值”、“地理坐标(GIS)”、“行军路线”。

输出物：纯文本格式的《史实摘要》(Fact Sheet)。无情报，不启动。

2. 指挥层 (Command Layer) —— [大脑]
执行者：Gemini 3 Pro (总监制)

核心职责：无限记忆中枢。利用超长 Context Window 记忆全剧本细节，负责由上至下的逻辑把控。
执行标准：
逻辑清洗：将 NotebookLM 提供的散碎史料，重组为符合“三段式架构”的叙事逻辑（势→法→果）。

指令封装：编写发给 DeepSeek 的标准化 Prompt，强制规定输出 JSON 格式。
明确界限：
    - 视觉逻辑（Prompt）：可以使用现代比喻（如“切断电源”指代切断补给）。
    - 最终文案（Script）：**严禁出现现代词汇（如 CPU、主板、电源）。**必须使用纯正的战国策/史记风（如“粮道绝”、“门户开”）。

3. 脚本层 (Scripting Layer) —— [指令]
执行者：DeepSeek V3 (金牌编剧)

核心职责：中文史诗文案与脚本生成。

执行标准：
1. 接收 Gemini 的大纲，撰写极具“中国历史味”的旁白。
2. 将文案与其对应的 GIS 坐标封装为 JSON 格式。

执行标准：输出双模态 JSON 脚本，驱动引擎运行：

Type A - 决策脚本 (AVG Schema)：

包含字段：background_id (场景), actor_asset (立绘), dialogue_text (旁白), bgm_track (音乐)。

Type B - 推演脚本 (MAP Schema)：

包含字段：spawn_unit (生成兵棋), move_path_coords (GIS路径坐标), camera_zoom (镜头高度), battle_fx (特效触发)。

4. 渲染层 (Rendering Layer) —— [演出]
执行者：Antigravity Engine (TypeScript/WebGL)
底层架构师：Claude 4.5 Opus (首席工程师)

核心职责：实时仿真与可视化。

核心能力：

混合状态机 (Hybrid State Machine)：支持在 AVG 模式（立绘演出）和 MAP 模式（地图仿真）之间进行帧级无缝切换，保证叙事连贯性。

实体管理 (Entity System)：

加载并渲染 SGZ11 (三国志11) 风格的高精度人物立绘与兵棋 Sprite。

实现基于贝塞尔曲线的平滑行军动画。

导演模式 (Director Mode)：

内置虚拟摄像机插值算法 (Lerp)，实现从 Zoom 7 到 Zoom 11 的丝滑运镜。

一键全屏运行，供 OBS 进行高码率录制。

第五部分：自动化技术工作流 (Technical Workflow v5.3)
1. 系统综述 (System Overview)项目代号：MAPWAR (Based on Antigravity Engine)核心定义：专为战史复盘设计的 “时空数据可视化引擎”。
技术哲学：Code-Driven (代码驱动)。拒绝传统非线编软件的手动拖拽，通过 TypeScript 事件编辑器实现“脚本加载 $\rightarrow$ 自动演算 $\rightarrow$ 实时渲染”。输出标准：4K / 60fps / 广播级动态动画。
2. 四层技术栈架构 (The 4-Layer Tech Stack)本系统执行严格的 “Data-to-Pixel” (从数据到像素) 单向流水线：Layer 1：情报层 (Data ETL)核心组件：NotebookLMIO：Raw History Text $\rightarrow$ Structured Fact Sheet职能：分段爆破。针对当前 Sequence 精准提取人物原话与 GIS 坐标。Layer 2：架构层 (Architecture)核心组件：Gemini (Pro/Ultra)IO：Fact Sheet $\rightarrow$ Standardized Prompt职能：逻辑中间件。决定“何时切换地图”、“何时使用特写”，生成发给 DeepSeek 的标准指令。Layer 3：脚本层 (Scripting)核心组件：DeepSeek (Coder)IO：Prompt $\rightarrow$ JSON Script职能：指令封装。生成引擎可读取的 JSON 代码，包含精确的时间戳与贝塞尔曲线参数。Layer 4：渲染层 (Rendering)核心组件：Antigravity Engine (TypeScript + WebGL)IO：JSON Script $\rightarrow$ Pixel Stream (OBS)职能：实时仿真。解析 JSON，驱动 Sprite 移动、相机运镜与特效粒子。
3. 核心运行逻辑：混合状态机 (Hybrid State Machine)引擎核心是一个无限循环的 Event Loop，根据 JSON 指令在三种状态间切换：State A (AVG)：文戏模式。静态背景 + 动态立绘 + 字幕。State B (MAP)：武戏模式。GIS 瓦片地图 + 兵棋 Sprite + 粒子特效。State C (UI)：结算模式。全屏高斯模糊 + 数据仪表盘。
4. 标准化数据协议 (JSON Protocol Standard) 所有 DeepSeek 生成的脚本必须遵循 EP_Schema.json 标准 (引入 Zod 严格校验)，包含 sequence_id, assets_preload, timeline 三大核心字段。
5. 工业化执行标准：7步作业法 (The 7-Step Protocol)



Pre-Step 1. 事件拆分与定调 (Segmentation & Hook)

执行：Gemini 3 Pro

动作：读取史料，拆解战役阶段，提炼“自媒体爆点” (如“消失的秦军”)。

产出：《分集策划案》 (Episode Plan)。

把每一集的各段按以下执行。

Step 2. 情报爆破 (Data ETL)执行：NotebookLM动作：仅查询当前段落史料（例如战争史，军事史，资治通鉴等）。产出：纯文本《史实摘要》(含 GIS 坐标与名言，数据)。

Step 3. 逻辑架构 (The Bridge)执行：Gemini动作：将摘要转化为结构化指令。产出：Standardized Prompt（包含运镜逻辑与叙事节奏）。

Step 4. 双模脚本生成 (Code Gen)执行：DeepSeek (Coder)动作：依据指令写代码。产出：script.json (包含 dialogue 字段，无需单独 txt)。让 DeepSeek 将四字军事术语（兵临城下、长驱直入、分兵合围、背水一战、弃甲曳兵、全军覆没）直接写入 JSON 对话。

Step 5. 资产并行生产 (Assets) 执行：Asset Bank (复用库) / Midjourney (固定微调模型) / TTS 动作：优先复用通用素材，缺损部分按 JSON 里的 assets_list 生产。产出：统一画风的立绘 Sprite、对齐文件名的语音 MP3。

Step 6. 引擎一键装配 (Assembly) 执行：Antigravity Engine 动作：将 JSON 和素材放入文件夹，热加载启动。产出：可实时预览的动态画面 (开启 Director Mode 拖动时间轴微调)。

Step 7. 导演模式采集 (Capture)执行：OBS动作：引擎开启无 UI 全屏模式，高码率录制。产出：4K 原始视频素材 (Raw Footage)。

Step 8. 灵魂注入 (Soul Injection)执行：剪映 / PR动作：合成声画。产出：添加物理音效（马蹄/兵器声）与史诗 BGM，最终成片。
