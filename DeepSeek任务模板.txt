1. 核心接口定义 (TypeScript Interface)
在你的代码中，你需要定义这几个基础类型。这就是给 DeepSeek 下指令时的“语法规则”。

TypeScript

// 定义所有支持的事件类型
type EventType = 
  | 'AVG_SET_BG'       // 1. 先背景
  | 'AVG_PLAY_VOICE'   // 2. 然后旁白/配音
  | 'AVG_SHOW_ACTOR'   // 3. 然后立绘
  | 'MAP_SWITCH'       // 4. 切换地形图
  | 'MAP_CAMERA'       // 5. 放大/运镜 (Zoom 9/10)
  | 'MAP_UNIT_MARCH'   // 6. 行军 (SGZ10素材移动)
  | 'MAP_BATTLE_FX'    // 7. 交战 (特效/整军)
  | 'UI_SHOW_RESULT';  // 8. 结果结算

// 每一个事件块的标准结构
interface GameEvent {
  id: string;          // 事件ID
  type: EventType;     // 事件类型
  params: any;         // 具体参数 (不同的类型参数不同)
  next_delay?: number; // 执行完等待几秒进下一个
}

// 整个脚本就是一串事件列表
interface EpisodeScript {
  title: string;
  events: GameEvent[];
}
2. 标准化流程演示 (JSON Payload)
这是EP01 第一段在你的编辑器里最终长成的样子。DeepSeek 以后就是负责生成这个 JSON。

流程： 背景 -> 旁白 -> 立绘 -> 切换 -> 运镜 -> 行军 -> 交战 -> 结果
