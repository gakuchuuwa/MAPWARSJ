# 剧情演出视觉规范 (Cinematic Event UI)

本文档定义了游戏中历史事件演出（Visual Novel 风格）的视觉与交互规范。

## 1. 核心设计理念

*   **沉浸感 (Immersion)**：通过背景模糊、暗角和遮罩，将玩家视线从战略地图强制引导至剧情中心。
*   **高级感 (Premium)**：采用“黑金”配色、衬线字体（Noto Serif SC）和精致的边框阴影。
*   **层叠感 (Depth)**：利用立绘“破框”效果和多层投影，建立强烈的空间纵深。
*   **聚焦 (Focus)**：居中构图，确保视觉平衡。

## 2. 布局规范 (Layout)

### 2.1 容器位置
*   **垂直位置**：UI 整体向上平移，底部 padding 约为 `10vh`。视觉重心位于屏幕中上部（黄金分割区域）。
*   **水平居中**：主容器使用 `justify-content: center`，左右 padding 保持对称（推荐 `5%`），确保对话框严格居中。

### 2.2 结构划分
*   **左侧立绘区**：宽 300px，用于主要说话人（如秦王）。
*   **中间内容区**：自适应宽度（max 1000px），包含对话框、名字条和背景插图。
*   **右侧立绘区**：宽 300px，用于对手戏角色（预留，目前隐藏但占位以保持平衡）。

## 3. 视觉元素 (Visual Elements)

### 3.1 背景处理 (Background & Atmosphere)
*   **地图模糊**：主容器应用 `backdrop-filter: blur(8px)`，模糊底层战略地图。
*   **全局压暗**：主容器背景色 `rgba(0,0,0,0.3)`，提升文字可读性。
*   **顶部暗角 (Vignette)**：使用固定在顶部的 `40%` 高度黑色渐变 `linear-gradient(to bottom, rgba(0,0,0,0.9), transparent)`，遮挡地图远景/地平线，消除杂乱细节。

### 3.2 插图样式 (Event Illustration)
*   **位置**：绝对定位居中（`top: 45%`, `left: 50%`）。
*   **边框**：1px `rgba(255, 215, 0, 0.3)` 金色边框。
*   **投影**：深色弥散投影 `box-shadow: 0 10px 40px rgba(0,0,0,0.8)`。

### 3.3 立绘表现 (Character Portraits)
*   **尺寸**：增强气场，最大高度 `550px`。
*   **镜像 (Mirroring)**：剧情对话 UI（`CinematicDialogueUI`）左立绘可固定 `scaleX(-1)`。**战斗 UI** 见 `portrait_defaults.ts` + `CombatUI` 朝向层，按文化区 PNG 朝向计算，勿套用「左攻一律镜像」。
*   **破框效果 (Break-out)**：
    *   立绘容器设置 `z-index: 10`（高于对话框）。
    *   立绘容器设置**负边距**（如 `margin-right: -50px`），使人物边缘覆盖在对话框边框之上。
*   **投影**：人物自身投影 `filter: drop-shadow(5px 5px 10px rgba(0,0,0,0.5))`。

## 4. 交互规范 (Interaction)

*   **全屏点击**：整个屏幕（任何区域）均为点击热区，点击即可推进剧情。
*   **推进提示**：右下角显示呼吸动画的“▼”图标。
*   **自动隐藏**：进入剧情模式时，自动隐藏游戏HUD（如小地图、日期栏）；退出时自动恢复。
