# 游戏底层机制备忘录：战斗、行军与特殊文化加成

本文档汇总了底层代码中关于行军速度、兵力上限、文化战力系数等机制，避免后续查阅源码时产生遗忘或混淆。

## 1. 军团行军速度机制 (`MARCH_SPEED_MULTIPLIERS`)

游戏中的行军速度并不是全局恒定的，而是严格受 **地形** 与 **文化基因**（是否为骑兵文化）共同影响。

### 1.1 地形基础倍率
- **山地 (mountain)**：`1.0`（基准速度）
- **平原 (plain)**：`1.5`
- **水域 (sea)**：`1.2`（注：帆船虽然绝对速度快，但受限于风向、靠岸、昼行夜泊，综合推进速度慢于平原行军）

### 1.2 纯骑兵文化专属加速
**适用对象**：草原 (STEPPE)、青藏 (TIBET)、西域 (WESTERN) 的纯骑兵三角阵型文化。
骑兵文化在陆地上移动时，会乘上专属的骑兵倍率（代码默认为 `current` 档位）：
- **平原骑兵加成**：`2.0`
- **山地骑兵加成**：`1.5`

**最终速度结论**：
- **普通步兵走平原**：`1.5（地形）× 1.0（无骑兵加成）= 1.5`
- **草原骑兵走平原**：`1.5（地形）× 2.0（骑兵加成）= 3.0`（比步兵快 100%！）
- **普通步兵走山地**：`1.0（地形）× 1.0（无加成）= 1.0`
- **草原骑兵走山地**：`1.0（地形）× 1.5（骑兵加成）= 1.5`（比步兵快 50%！）

## 2. 兵力上限限制 (`TRIANGLE_CAVALRY_ARMY_MAX_TROOPS`)

虽然骑兵文化（草原、青藏、西域）拥有极高的机动性和野战加成，但底层在兵力上限上做出了严厉的限制：
- **普通方阵文化（如中原）军团兵力上限**：`100,000`（10 万）
- **纯骑兵三角阵文化军团兵力上限**：`70,000`（7 万）

这逼迫游牧势力必须利用高机动性进行“狼群战术”以多打少，而不能与中原大国硬拼单军团规模。

## 3. 文化战力乘区与据点加成

在真实的战力计算中，所有的系统乘区**都会 100% 参与底层运算**（即便在 UI 上因为系数为 1.0 而被折叠隐藏）。

### 3.1 进攻方与防守方的判定 (Field vs Garrison)
- **城防 (Garrison)**：只有当据点被直接攻打时，据点内的防守部队才视为“城防”。
- **野战 (Field)**：**任何主动出征的军团**，无论是去打纯野战，还是去**攻打别人的城池**，全部套用“野战”系数。进攻方不吃亏。

### 3.2 典型文化系数
- **高攻脆皮组（草原、青藏、西域、河西等）**：
  - 野战系数：`×1.2` 或 `×1.1`
  - 城防系数：`×0.8` 或 `×0.9`
- **铁壁防守组（岭南、滇黔、巴蜀等）**：
  - 野战系数：`×0.8`
  - 城防系数：`×1.2`
- **均衡组（中原、中亚等）**：
  - 双系均为 `1.0`

### 3.3 拒险而守：关隘加成
如果一座城市在底层数据中的 `type` 为 `pass`（关隘），则在防守时会额外获得一个独立的 **拒险而守系数（×1.2）**。
- *例：岭南的关隘被攻击，防守方将获得 `1.2（岭南文化）× 1.2（关隘天险）= 1.44` 的叠加防守系数。*

## 4. 战斗 UI 面板展示逻辑

UI 面板的 `formatBattlePowerFactorChain` 方法负责展示长串的战力公式链条（如 `0.9 × 1.1 × 1.5`）。
- **极简折叠机制**：只要某个乘区计算结果正好为 `1.0`（误差在 `0.001` 内），面板就会将其隐藏。
- 因此，未显示的加成并非“失效”，而是恰好是不增不减的平局系数（`1.0`）。
