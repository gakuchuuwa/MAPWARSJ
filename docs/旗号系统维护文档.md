# 🚩 旗号（旗帜文字）系统维护文档

> **目标读者**：后续维护该项目的 AI 或开发者
> **核心问题**：添加新势力后，旗帜上不显示文字 / 显示英文 / 显示为空白
> **最后更新**：2026-05-28

---

## 一、概述

每个势力在游戏地图上显示一面旗帜（flag），旗帜上方渲染该势力的中文简称（"旗号"）。旗号文字通过 Canvas 动态生成 Data URL，作为背景图显示在 DOM 元素上。

旗号显示完整的**数据链路**：

```
factions.ts (势力定义)
    ↓
cities_v2.ts (城池数据，factionId 关联势力)
    ↓
GameApp.ts loadCityData() ──► SANDBOX_MODE 检查 ──► 决定最终 factionId
    ↓
TerritorySystem.renderSingleCity() ──► 创建 flagTextOverlay div
    ↓
CityAssetManager.getProcessedFlagText(factionId)
    ├── 1. 查 sandboxDisplayNames (SandboxDisplayNames.ts)
    ├── 2. 查 _legacy_sandbox_dict (CityAssetManager.ts 内部)
    ├── 3. 查 DynamicFlagTextGenerator.generate() 生成 Canvas
    └── 4. 缓存到 processedFlagCache (key: "dynamic_text_{factionId}")
    ↓
CSS 显示 .city-flag-body 背景图
```

## 二、添加新势力的 6 个必注册点

**每次添加新势力 + 据点时，必须同时修改以下 6 个文件，缺一不可。**

### ① `src/data/factions.ts` — 势力定义

```typescript
{ id: 'she', name: '奢', color: '#8B0000' },
//  ├── id: 势力唯一标识符 (全小写英文)
//  ├── name: 显示名称（中文）
//  └── color: 旗帜底色（十六进制）
```

### ② `src/data/cities_v2.ts` — 城池数据

在 `CITIES_V2` 数组末尾（或 `PERIPHERY`/`RESTORED_CITIES` 中）添加城池：

```typescript
{ id: 'city_yongning2', name: '永宁', factionId: 'she',
  lat: 28.17, lng: 105.44, type: 'small_city', troops: 10000,
  tier: 2, note: '...' }
```

> ⚠️ `tier` 只允许值：`0 | 1 | 2 | 4`（没有 3 ！）
> ⚠️ `type` 必须是 `CityType` 联合类型中的有效值

### ③ `src/data/SandboxDisplayNames.ts` — 旗号文字（首要来源）

```typescript
'she': '奢',
```

这是旗号文字的**首选来源**。`getProcessedFlagText()` 优先从这里查找。

- 值通常为 1~2 个中文字（旗帜空间有限）
- 单姓：`'she': '奢'`
- 复姓/双字：`'gongsun_d': '公孙'`
- 长名截断：`'tushetu': '土谢'`（取前二字）

### ④ `src/core/CityAssetManager.ts` — `factionFlagMap`

在 `factionFlagMap` 静态映射表末尾添加：

```typescript
'she': 'RANDOM',
```

- `'RANDOM'`：随机分配一面旗帜图案
- 指定路径：`'url("path/to/flag.png")'`：使用特定旗帜图

### ⑤ `src/core/CityAssetManager.ts` — `_legacy_sandbox_dict`

在 `getProcessedFlagText()` 方法内的 `_legacy_sandbox_dict` 对象末尾添加：

```typescript
'she': '奢',
```

> **为什么有两份？** `_legacy_sandbox_dict` 是早期遗留的备份映射，`getProcessedFlagText()` 在查完 `sandboxDisplayNames`（外部）后会回退到 `_legacy_sandbox_dict` 查找。两者必须保持一致，否则会导致"某些情况下有字、某些情况下没字"的怪异 bug。

### ⑥ `src/core/GameApp.ts` — `STARTING_CAPITALS`

```typescript
'she': 'city_yongning2',
```

**这是最关键、最容易遗漏的一步！**

## 三、SANDBOX_MODE 陷阱（核心！）

### 症状

旗帜图案正常显示，但旗帜上**没有文字**（或显示英文 factionId）。

### 根因

[`GameConfig.ts:8`](../src/config/GameConfig.ts:8) 中 `SANDBOX_MODE = true`。

当 `SANDBOX_MODE` 开启时，[`GameApp.loadCityData()`](../src/core/GameApp.ts:972) 会对每个城池执行以下逻辑：

```typescript
// 简化伪代码
const isCapital = STARTING_CAPITALS[factionId] === c.id;
if (!isCapital) {
    const ownerFaction = Object.keys(STARTING_CAPITALS)
        .find(key => STARTING_CAPITALS[key] === c.id);
    if (ownerFaction) {
        factionId = ownerFaction;  // 该城池被其他势力作为都城占据
    } else {
        factionId = 'panjun';     // ❗ 重置为叛军！
    }
}
```

**后果**：任何未在 `STARTING_CAPITALS` 中注册的城池，其 `factionId` 都会被强制改为 `'panjun'`（叛军）。

而 [`CityAssetManager.getProcessedFlagText()`](../src/core/CityAssetManager.ts:794) 中：

```typescript
if (!factionId || factionId === 'panjun') return null;  // ⬅️ 叛军无旗号文字！
```

所以旗子上没有字。

### 解决方案

在 `STARTING_CAPITALS` 中为新势力的据点注册：

```typescript
'she': 'city_yongning2',
//  factionId : 城池ID
```

表示："she 势力的首都是 city_yongning2"。这样 `loadCityData()` 就会保留 city_yongning2 的 factionId 为 `she`。

### 多个城池怎么办？

如果该势力有多个城池，只有**首都能保留 factionId**，其余城池也会被重置为 `panjun`。这是因为 sandbox 模式下每个势力只保留一座都城。

## 四、旗号文字渲染流程详解

### 4.1 入口

`TerritorySystem.renderSingleCity()` → 为每个城池创建 `flagTextOverlay` div。

### 4.2 文字获取

[`CityAssetManager.getProcessedFlagText(factionId)`](../src/core/CityAssetManager.ts:794)：

```
1. 检查 factionId 是否为 null / undefined / 'panjun'
   → 是 → 返回 null（不显示文字）

2. 查缓存 processedFlagCache (key: "dynamic_text_{factionId}")
   → 命中 → 返回缓存的 Data URL

3. 查 sandboxDisplayNames（来自 SandboxDisplayNames.ts）
   → 找到 → 用该文字
   → 没找到 → 查 _legacy_sandbox_dict（内部备份）
   → 还没找到 → 从 factionManager.getFactionName() 获取势力全名
   → 还是没找到 → 用 factionId 本身（显示英文）

4. 根据旗帜亮度选字色：
   - 浅色旗 (lum >= 128) → 黑色字 + 白色描边
   - 深色旗 (lum < 128)  → 白色字 + 黑色描边

5. 调用 DynamicFlagTextGenerator.generate(text, fill, stroke)
   → 生成 Canvas Data URL

6. 存入缓存，返回 Data URL
```

### 4.3 Canvas 渲染

[`DynamicFlagTextGenerator.generate()`](../src/utils/DynamicFlagTextGenerator.ts:12)：

- 画布尺寸：`4 × renderScale`（通常 32×40 每帧）
- 4 列 × 6 行 = 24 帧动画（旗帜飘动）
- 双字垂直排版：两字上下排列
- 字号：`8px` 粗体
- 输出：Data URL（PNG）

### 4.4 CSS 显示

```css
.city-flag-body {
    animation: cityFlagWave 0.6s steps(1) infinite;
}
```

`flagTextOverlay` div 使用 `background-image` 设置为 Data URL，通过 CSS 动画逐帧播放。

### 4.5 缓存

旗帜文字 Data URL 缓存在 `processedFlagCache` 中，key 格式为 `dynamic_text_{factionId}`。**强制刷新浏览器 (Ctrl+Shift+R) 可清除缓存。**

## 六、调试方法

如果某个势力的旗号不显示，按以下顺序排查：

### 6.1 检查浏览器控制台

```javascript
// 查看势力 ID 是否被重置为 panjun
game.factionManager.getFactionByCity('city_duolun')

// 手动获取旗号文字
CityAssetManager.getProcessedFlagText('chahar')
```

### 6.2 检查 `_legacy_sandbox_dict`

```javascript
// 在 CityAssetManager.ts getProcessedFlagText() 中
// 在 return null 之前加调试日志：
console.log(`[FLAG_DEBUG] factionId=${factionId}, textToRender=${textToRender}`);
```

### 6.3 检查 STARTING_CAPITALS

```javascript
// GameApp.ts 的 loadCityData() 中
// 在 factionId 被重置前打印日志：
console.log(`[SANDBOX] city=${c.id}, faction=${factionId}, isCapital=${isCapital}`);
```

### 6.4 调试工具

[`src/utils/FlagTextDebug.ts`](../src/utils/FlagTextDebug.ts) 中有一个完整的调试函数：

```typescript
// 在浏览器控制台调用
debugFlagText('chahar');
// 输出：是否在 sandboxDisplayNames 中找到 →
//       是否在 _legacy_sandbox_dict 中找到 →
//       最终使用的文字 →
//       旗帜亮度值 →
//       生成的 Data URL（可点击预览）
```

### 6.5 缓存问题

如果修改了旗号文字但浏览器没刷新：

1. **Ctrl+Shift+R** 强制刷新（跳过缓存）
2. 或在控制台执行 `CityAssetManager.processedFlagCache.clear()`
3. 或修改代码后等待 Vite HMR 自动重载

## 七、典型错误案例

### ❌ 错误：只添加了 factions.ts 和 cities_v2.ts

结果：城池出现在地图上，但 factionId 被 sandbox 模式重置为 `panjun`，旗帜显示叛军图案且无文字。

### ❌ 错误：忘了 _legacy_sandbox_dict

结果：有时有字有时没字。`sandboxDisplayNames` 和 `_legacy_sandbox_dict` 必须同时更新。

### ❌ 错误：用了 tier: 3

结果：TypeScript 编译错误，CityDataV2 的 tier 类型为 `0 | 4 | 1 | 2`，不支持 3。

### ❌ 错误：更新了代码但没强制刷新浏览器

结果：`processedFlagCache` 缓存了旧 Data URL，需要 Ctrl+Shift+R 清除。
