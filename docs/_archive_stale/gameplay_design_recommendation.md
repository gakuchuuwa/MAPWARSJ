# 《历史大乱斗 (MAPWAR)》玩法设计推荐文书

> 版本：v1.0（2026-06-10）  
> 性质：原玩法方案详细版；方向基准请以 GAME_DIRECTION.md 为准。

## 第一章 对本作的根本理解

### 1.1 这是什么游戏

一部**基于真实地理与严谨史实的世界地形推演模拟器**。历史教材属性优先于娱乐属性，所有游戏性必须在历史严谨性的镣铐下生长。

### 1.2 三大红线（任何方案的先决审查项）

1. **历史真实性**：据点坐标、势力名称、旗号必须史料可考（民族 > 政权 > 郡治 > 家族 > 州名五级定旗）。宁缺毋滥，禁止虚构。
2. **一城一势一旗（开局公理）**：1 势力 = 1 据点 = 1 旗号 = 1 现役军团 = 1 唯一 HSL 主色。**这是开局状态约束，不是全程上限**——攻城易帜、征服扩张是核心玩法（`SiegeManager` 现行逻辑即转移 `factionId`）。
3. **地理约束**：据点间距 ≥ 50 km；坐标锚定真实史地，禁止为机制挪点。

### 1.3 红线反转为卖点（设计总纲）

| 红线 | 反转卖点 |
|---|---|
| 一城开局 | **每一面旗都是一条命**，每次出征都是国运豪赌 |
| 史实据点 | **地图本身就是攻略本**，历史知识即战力（懂汉中是川蜀门户者天然占优） |
| 确定性结算 | **胜负在出征前已写好**，深度全部前置到战略准备期，是棋的博弈而非动作的操作 |

### 1.4 既有系统资产（设计必须长在其上）

`TerritorySystem`（格网 BFS 领土）、`SiegeManager` / `LegionFieldBattle`（确定性攻城/野战）、`UnitCounterSystem`（步骑弓三角克制）、`FollowResupplySystem`（随军补给）、`RoadRegistry` / `LegionRoadMarch`（史实道路网行军）、`RebellionSystem`（按文化区年度叛乱）、`RegionSystem`（14 区 × 15 文化中心）、`HistoricalEventManager`（历史事件管线）、20 大城白名单。

### 1.5 关键澄清（防止后续方案误读）

- 本作是**跨时代大乱斗**（秦、汉、大清、德川同图），**不**锚定任何单一历史年份快照。
- 领土是格网 BFS 增量计算，**不是**解析 Voronoi；全图重算曾致主线程卡死（AGENTS.md §10.1），任何"领土随军实时流动"类方案必须走局部增量。
- 行军走**人工考据的史实道路网**，这是项目资产；禁止用自动生成路径（Delaunay 中轴线等）替代。
- 伪直播架构以**观赏性**为前提，地形与版图全公开；不做全图深迷雾。

## 第二章 战斗与后勤数学内核（采纳自外部方案，已过红线审查）

### 2.1 兰彻斯特双律动态切换 ⭐ 最高优先

按交战地形的展开面**切换结算方程**，而非堆叠修正系数：

- **狭窄地形**（渡口、峡谷、密林，pass 类据点附近）→ **线性律**：\(\frac{dA}{dt} = -\alpha AB\)。兵力数量优势被降维，单兵效能（兵种克制、武将统率）主导 → "一夫当关"成为方程自然解。
- **开阔地形**（平原、草原）→ **平方律**：\(\alpha(A_0^2 - A_t^2) = \beta(B_0^2 - B_t^2)\)。战斗力与兵力平方成正比，劣势兵力平原接战 = 被迅速全歼 → 逼出"集中兵力、分割机动"的真实战术。

平方律有闭式解，指令下达瞬间即可算出终局时间戳与逐日战损序列——**与伪直播架构完全同构**。

### 2.2 Helmbold 攻城模型

攻城战引入防守方优势参数 + 苦涩度参数，守方胜率经逻辑斯谛函数得出。设计后果：**兵力对比在攻城战中是劣质预测指标**，逼玩家走围困断粮、多路合围，而非堆人头。挂接 `SiegeManager` 现有框架。

### 2.3 补给摩擦与断粮崩盘

- 军团粮耗按脚下色块分档（本方 ×1.0 / 无主 ×1.5 / 敌方 ×2.5），深入敌境距离 \(D\) 附加指数项 \(C = C_{base} \cdot e^{kD}\)。
- 敌境行军 = 路径对敌全程可见（深入即暴露）。
- 归途色块连通被切断 → 判定**断粮**：全图叙事播报「X 军粮道断于 Y 地」，其兰彻斯特效能系数 \(\beta\) 在数日内几何级数衰减。**后勤与战斗用同一方程焊死**——长平、官渡的历史逻辑由系统涌现，无需写剧本。
- 军团头顶常驻粮袋图标（满/半/瘪三档），断粮状态变色，逼玩家在"强攻速胜"与"回师保命"间抉择。
- 地基：`FollowResupplySystem` + `TerritorySystem` 连通性判定。

### 2.4 兵种与地形二维修正

保留 `UnitCounterSystem` 三角克制；叠加地形项：骑兵平原野战 +30%、入山林 -40%；渡口野战渡河方首日 -50%（"半渡而击"可执行）。**全部修正在出征前路径预览中明示**（预计天数/粮耗/风险着色）——深度来自规划，不来自隐藏数值。

> **明确拒绝**：秒伤密度分布函数（非均匀军队逐秒积分求导）——复杂度暴涨而玩家不可感知，过度设计。

## 第三章 单城内政——「都城四仓」与「军团成色」

> **【已废弃 · 观察型定位】** 本章为玩家操作型内政，不再作为开发方向。

### 3.1 设计思路：不养城，养军

单城内政收敛为一个问题：**你这唯一一支军团，出征时是什么成色？** 城内只设 4 个抽象数值仓，不画建筑：

| 仓 | 来源 | 去向 | 博弈点 |
|---|---|---|---|
| **粮** | 领土面积 × 地形产出系数 | 出征按里程预扣 | 领土被蚕食 = 不打仗也会被掐死 |
| **丁** | 城池 tier + 和平年数 | 募兵、补战损 | 全歼后需数年恢复 → 野战代价极重；**募兵抽走劳力 → 粮产下降**（穷兵黩武负反馈，挂 `RebellionSystem`） |
| **械** | 时间 + 文化区科技倾向（草原出骑、中原出弩，挂 `RegionSystem`） | 决定可编兵种配比 | 与三角克制联动 |
| **望** | 守土、胜仗、占文化中心 | 战败、背盟、断都 | 过低 → 本城起义易帜 |

四仓不可兑换，同一时刻只能"重点积累"一仓——单城没有空间纵深，用**时间纵深**制造选择压力。

### 3.2 编军 = 出征前的一次性配装

出征瞬间决策（之后零微操，纯指令驱动）：带多少丁、步骑弓配比（吃械仓、针对情报配克制）、携粮天数（轻装疾行 vs 重辎缓进，喂给 `FollowResupplySystem`）。一次出征 = 一道配装谜题，**算得准的人赢**。

## 第四章 外交与情报——契约、借刀与成色迷雾

> **【已废弃 · 观察型定位】** 本章为玩家外交/情报 UI，不再作为开发方向。

### 4.1 三个外交指令（全文本指令，零带宽）

| 指令 | 效果 | 背叛代价 |
|---|---|---|
| **会盟** | 盟方色块内免补给惩罚 + 共享视野 | 望仓暴跌 + 全图广播「XX 背盟」 |
| **借道** | 准许一次穿越路径 | 中途倒戈 → 触发"假道伐虢"事件，望清零 |
| **称臣纳贡** | 弱方年输粮，强方被攻必援（AI 强制） | 强方见死不救 → 全部附庸自动解约 |

不设好感度数值——确定性世界里，**履约记录本身就是数值**。

### 4.2 情报：只迷雾"敌军成色"

地形与版图全公开（保观赏性），迷雾只罩一样东西：**敌军兵种配比与粮储**。默认只见旗号 + 规模档位（"约万人"）；派单骑斥候侦明成色（可被敌境捕获，捕获即打草惊蛇）。高级玩法：开放"示形"指令主动泄假成色。确定性结算 + 不完全信息输入 = 棋手式博弈。

### 4.3 征服后处理：占领 / 招降二选一

攻城胜利后选其一：

- **占领**：易帜转 `factionId`（现行机制）；
- **招降为附庸**：城池保留原旗号与原色（**不做同色渐变，恪守一势一色**），周期纳贡输粮，距宗主越远忠诚衰减越快，宗主主力陷苦战时易叛乱复独立。

## 第五章 涌现叙事——天命、连锁与流亡

### 5.1 天命 = 仇恨值（反滚雪球自平衡器）

- 占本区文化中心 → 区内望仓+，panjun 叛乱率降；
- 跨区占大城（如草原势力占洛阳）→ 「入主中原」叙事链：望大涨，但年年"治理判定"，失败即起义；
- 同时据长安 + 洛阳 → 全图广播「天命所归」，**所有未结盟 AI 转为戒惧、倾向合纵抗你**。天命不是胜利点数，是包围网触发器。

### 5.2 三事件连锁语法

```
攻城(洛阳陷) → 叙事(迁都/望崩) → 野战(勤王军虎牢相撞)
叙事(史实灾年大旱) → 区域粮仓-50% → AI 被迫出征就食 → 攻城
野战(全歼) → 叙事(坑杀/收编 二选一) → 望与丁的取舍
```

灾异事件**只取史实灾年记录**（旱蝗河决，可考才入库）——教材属性即随机池质检标准。气候驱动游牧南下的触发思路可用，但不锚定具体年份模型。

### 5.3 亡国不退场：流亡机制（史料版）

都城陷落时，若该政权**史载有可考迁都地**且该点现为 panjun → 触发「衣冠南渡」，旗号迁往该点复国（四仓减半）；无可考迁都地 → 真亡国。**严禁"随机逃亡到附近城镇"**——迁都表必须逐势力考据建档。历史上真迁过都的政权游戏里多一条命，再次"知识即战力"。

## 第六章 心流与表现——确定性结算的史诗包装

| 层级 | 手法 |
|---|---|
| 战略层（洲级） | 军情室美学：等值线 + 呼吸色块 + 出征光线沿史实道路流动；深沉底噪 + 落子般"咔哒"声 → 君王级掌控感 |
| 战役层（省州级） | 相撞 0.3s 顿帧 + 轻震屏；原位粒子爆发（**不切场景**）；逐日跳动的红蓝伤害数字瀑布，劣势方阵型粒子渐溃散暗淡 |
| 播报层 | 重大事件竖排书法字幕横扫（「官渡，袁军粮道断」）+ 金石音效 |

**核心技巧——确定性结算 + 分段式播放**：后台瞬间算完终局，前台按"日"逐段播放战损，且允许播放中途下达"援军/撤退"指令作为下一次结算输入。观感是直播，架构仍是纯指令驱动。

可选彩蛋：史诗级歼灭战在地表留**永久焦痕纹理**（表现层贴花，不污染真实地貌渲染管线）。

## 第七章 红线否决清单（后续方案审查用）

以下类型的提案**一律否决**，无需讨论：

1. 锚定单一历史年份的开局快照（本作是跨时代大乱斗）；
2. 自动生成寻路网（Delaunay/中轴线）替代史实道路网；
3. 军团作动态生成点触发全图领土重算（主线程卡死前科，只许局部增量）;
4. 全图深度战争迷雾（杀死伪直播观赏性）；
5. "随机"选择迁都地/新生据点/凭空分裂微型据点（违反史料优先，新势力只能由既有 panjun 点易帜承接）；
6. 附庸渲染为宗主同色系渐变（违反一势一色）；
7. 逐秒积分求导级的战斗微观模型（过度设计）；
8. 任何要求挪动据点坐标、虚构地名以迁就机制的设计。

| 优先级 | 模块 | 理由 |
|---|---|---|
| **P0** | §2.1 双律切换 + §2.3 粮道断粮 | 结算内核 + 战略深度，地基最全（补给/领土/道路俱在） |
