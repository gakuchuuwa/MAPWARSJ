# 战斗系统说明

## 统一战斗逻辑

游戏中所有战斗都使用统一的战斗逻辑，详见 `game.js` 中的 `resolveSiege` 函数注释。

### 战斗公式

```
胜方剩余 = 原兵力 - (败方兵力 × 随机(0.6-0.8) / 2)
败方剩余 = 0（全灭）
```

### 说明

1. **胜负判断**：兵力多的一方获胜（相等时守方胜）
2. **伤亡计算**：胜方伤亡 = 败方兵力 × 0.6~0.8 / 2
3. **伤兵恢复**：除以2相当于50%伤兵恢复
4. **败方结果**：败方全灭

### 示例

```
15000 vs 8000
- 胜方：15000
- 败方：8000（全灭）
- 伤亡：8000 × 0.7 / 2 = 2800
- 剩余：15000 - 2800 = 12200
- 伤亡率：18.7%
```

### 平衡性调整

- **增大0.6-0.8范围** → 战斗更残酷
- **减小0.6-0.8范围** → 战斗更温和
- **修改/2系数** → 调整恢复比例

## 适用范围

此逻辑适用于所有战斗类型：

- ✅ AI攻城
- ✅ 玩家攻城
- ✅ 玩家打NPC
- ✅ 历史事件攻城
- ✅ 历史事件野战
- ✅ 玩家参与任何战斗

## 历史事件配置

### ⚠️ 已废弃的配置

`historicalEvents.js` 中的 `casualtyRatio` 配置已不再使用，可以忽略或删除。

历史事件中的战斗现在也使用统一的战斗逻辑，不再使用自定义的伤亡比例。

### 历史事件战斗配置

历史事件中只需要配置：

```javascript
{
    type: "siege",
    attackerFactionId: "qin",
    targetCityId: "zheng_zhou",
    duration: 15000  // 战斗持续时间（毫秒）
    // casualtyRatio 已废弃，不再使用
}
```

## AI战斗系统

### AI出兵条件

```
1. 城市兵力 ≥ 10000
2. 势力总兵力 ≥ 100000
3. 距离上次出兵 ≥ 1年
4. 出兵兵力 ≥ 敌方兵力 + 10000
```

### AI目标选择

```
- 搜索范围：300公里
- 目标类型：仅敌方城市（不攻打NPC）
- 优先级：距离最近的符合条件目标
```

### AI出兵规则

```
- 出兵数量 = 城市兵力 - 5000
- 城市留守 5000 兵
- 最低出兵数量：5000兵
```

## 代码位置

- **主战斗逻辑**：`game.js` → `resolveSiege()` 函数
- **玩家攻城**：`game.js` → `resolvePlayerSiege()` 函数
- **玩家打NPC**：`game.js` → `resolveNPCBattle()` 函数
- **历史事件野战**：`game.js` → `updateArmies()` 函数中的野战部分
- **AI决策**：`game.js` → `executeAIDecisions()` 函数
