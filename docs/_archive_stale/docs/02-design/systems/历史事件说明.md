# MAPWAR 历史事件系统说明

## 📅 最后更新
2025年10月31日 19:45 (UTC+8)

---

## 🎯 概述

MAPWAR 游戏的历史事件系统允许在游戏进程中触发预定义的历史事件，这些事件会改变游戏状态，包括城市归属、军队调动、战斗等。

---

## 📋 事件结构

每个历史事件包含以下字段：

```javascript
{
    id: "event_id",              // 事件唯一标识
    year: -206,                   // 发生年份
    season: 0,                    // 发生季节 (0=春, 1=夏, 2=秋, 3=冬)
    title: "事件标题",            // 事件标题
    description: "事件描述",      // 事件详细描述
    actions: [...]                // 事件动作列表
}
```

---

## 🎬 支持的动作类型

### 1. spawn_army - 生成军队

生成一支军队从一个城市移动到另一个城市。

```javascript
{
    type: "spawn_army",
    factionId: "qin",              // 势力ID
    fromCityId: "chang_an",        // 出发城市（可选）
    toCityId: "handan",            // 目标城市
    autoSelectNearest: true,       // 自动选择最近的同势力城市作为出发点
    troops: 20000,                 // 军队数量
    speed: 1.0                     // 移动速度倍率
}
```

**说明：**
- 如果指定 `fromCityId`，军队从该城市出发
- 如果使用 `autoSelectNearest`，自动选择距离目标最近的同势力城市
- 如果势力没有城市，在目标城市附近生成军队

---

### 2. spawn_city - 动态创建城市 ⭐ 新功能

在游戏进行中动态创建新城市。

```javascript
{
    type: "spawn_city",
    cityId: "hanzhong",            // 城市唯一ID
    name: "汉中",                  // 城市名称
    factionId: "qi",               // 归属势力
    latitude: 33.07,               // 纬度
    longitude: 107.02,             // 经度
    importance: "capital",         // 城市重要性 (capital/major/minor)
    troops: 15000                  // 初始兵力
}
```

**说明：**
- 城市会立即出现在地图上
- 如果城市ID已存在，跳过创建
- 可以设置为都城（capital）、主要城市（major）或次要城市（minor）

---

### 3. faction_destroyed - 势力灭亡 ⭐ 新功能

标记一个势力灭亡。

```javascript
{
    type: "faction_destroyed",
    factionId: "qin",              // 灭亡的势力ID
    message: "秦国灭亡"            // 显示消息
}
```

**说明：**
- 记录势力灭亡事件
- 城市归属保持不变（由后续事件处理）
- 可用于历史记录和统计

---

### 4. transfer_city - 转移城市

将城市从一个势力转移到另一个势力。

```javascript
{
    type: "transfer_city",
    cityId: "chang_an",            // 城市ID
    fromFactionId: "qin",          // 原归属势力
    toFactionId: "chu",            // 新归属势力
    message: "项羽占领咸阳"        // 可选消息
}
```

**说明：**
- 如果事件中有战斗（siege 或 field_battle），城市转移会在战斗结束后执行
- 如果没有战斗，立即转移城市归属
- 城市兵力保持不变

---

### 5. siege - 攻城战

发起对城市的攻城战。

```javascript
{
    type: "siege",
    attackerFactionId: "chu",      // 攻击方势力
    targetCityId: "chang_an",      // 目标城市
    duration: 15000,               // 战斗持续时间（毫秒）
    casualtyRatio: {               // 伤亡比例
        defender: 1.0,             // 防守方伤亡倍率
        attacker: [1.1, 1.5]       // 攻击方伤亡倍率范围
    }
}
```

**说明：**
- 需要配合 `spawn_army` 使用，先生成攻城军队
- 战斗结束后，根据结果决定城市归属
- 伤亡比例影响双方损失

---

### 6. field_battle - 野战

在指定地点进行野战。

```javascript
{
    type: "field_battle",
    location: "handan",            // 战斗地点（城市ID）
    battleName: "巨鹿之战",        // 战斗名称
    factions: [                    // 参战势力
        { factionId: "qin", troops: 10000 },
        { factionId: "chu", troops: 10000 }
    ],
    duration: 15000                // 战斗持续时间（毫秒）
}
```

**说明：**
- 在指定城市附近生成双方军队
- 战斗结束后，失败方军队消失
- 可以有多个势力参战

## 🎮 事件执行流程

### 时间触发
- 游戏按照时间线推进（年份 + 季节）
- 每个季节结束时检查是否有事件触发
- 符合条件的事件按顺序执行

### 动作执行顺序
1. **spawn_city** - 立即创建城市
2. **faction_destroyed** - 记录势力灭亡
3. **spawn_army** - 生成军队并开始移动
4. **field_battle / siege** - 开始战斗
5. **transfer_city** - 战斗结束后转移城市（如有战斗）

### 战斗处理
- 攻城战和野战会显示战斗动画
- 战斗持续指定时间
- 战斗结束后根据结果执行后续动作


## 🔧 技术实现

### 城市动态创建
- 城市数据存储在 `GameManager.cities` 数组中
- 新城市会立即在地图上渲染
- 支持设置都城、主要城市、次要城市等级

### 势力管理
- 势力数据定义在 `gameData.js` 中
- 势力灭亡事件记录在日志中
- 城市归属通过 `factionId` 字段管理

### 事件触发
- 事件定义在 `historicalEvents.js` 中
- 时间线系统自动检查并触发事件
- 支持多个动作按顺序执行

---

## 📝 注意事项

1. **城市ID唯一性**
   - 每个城市必须有唯一的ID
   - 重复创建会被跳过

2. **势力ID一致性**
   - 确保 `factionId` 与 `gameData.js` 中定义的势力ID一致
   - 暂时用齐国（qi）代表汉王，后续可添加汉国势力

3. **战斗与城市转移**
   - 如果事件包含战斗，城市转移会在战斗结束后执行
   - 确保 `transfer_city` 动作在战斗动作之后

4. **兵力管理**
   - 生成军队会从势力总兵力中扣除
   - 如果兵力不足，会自动补全

5. **坐标准确性**
   - 城市坐标使用经纬度
   - 确保坐标在地图范围内
