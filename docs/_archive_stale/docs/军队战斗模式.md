# 军队战斗系统 (Strategic Combat System)

本文档描述当前游戏的战略地图自动战斗系统，包括寻路、接触判定与攻城逻辑。

> **历史说明**: RTS 微观战斗模式 (Zoom 12) 已于 2026/02 移除，当前系统专注于宏观战略层面的自动演算。

## 1. 核心设计理念
- **去格子化 (De-gridification)**: 战斗与移动不再依赖六边形网格 (Hex Grid) 的离散判定，而是基于连续的地理坐标 (Latitude/Longitude) 和欧几里得距离。
- **自动演算 (Auto-Resolution)**: 战斗结果由数值模型自动计算，无需玩家微操。
- **视觉反馈**: 在大地图上直接展示兵团动画（冲锋、射击、格挡），所见即所得。

## 2. 攻城战机制 (Siege Mechanic)

### 2.1 触发条件 (Trigger Condition)
攻城战的触发不再依赖于军队通过“格子邻接”判定，而是采用**距离阈值**检测。

- **触发距离**: `GameConfig.SIEGE.COMBAT_RADIUS` (默认 **0.15**，约合 16km)。
- **逻辑流程**:
  1. 军队向目标城市移动。
  2. `LegionManager` 在每帧更新时检测军队与目标城市的距离。
  3. 当距离 `<= 0.15 + 容差` 时，视为到达攻城阵地。
  4. 军队停止移动并切换至 `SIEGE` 状态，通知 `SiegeManager`开始战斗。

### 2.2 路径处理 (Path Trimming)
为防止军队“走进城市中心再弹回”的视觉抖动，系统实现了**路径预截断**。

- 当军队的目标是**敌对城市**时，寻路系统生成的路径会自动在距离城市中心 `COMBAT_RADIUS` 处截断。
- 军队会自然地停在城外指定的攻城阵地上，无需任何额外的位移修正。
- 如果是**友方城市**，军队则保留进入城市的路径（进驻/补给）。

### 2.3 战斗定位 (Battle Positioning)
- 攻城开始后，`SiegeManager` 会根据军队进入的角度，将其在视觉上锁定在城外的固定距离环上。
- 多个军团同时攻城时，会按照各自的进入角度分布在城市周围。

## 4. 视觉表现
- **兵团渲染**: 使用 `LegionPhalanxDrawer` 绘制大规模方阵。
- **状态动画**:
  - `MOVE`: 行军状态。
  - `ATTACK`: 攻击动作（近战/射击）。
  - `IDLE`: 待机/布阵。
