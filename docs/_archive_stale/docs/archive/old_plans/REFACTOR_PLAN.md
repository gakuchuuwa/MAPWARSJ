# MAPWAR 代码重构计划

## 目标
优化现有代码架构，提升性能和可维护性，为未来添加战斗系统做准备。

## 重构原则
1. **渐进式**：每一步都保持项目可运行
2. **向后兼容**：保留所有现有功能
3. **性能优先**：优化渲染和计算性能
4. **模块化**：清晰的代码结构

## 关键优化点

### 1. 渲染优化
**问题：** 每帧都重绘整个地图
**解决：** 
- 离屏 Canvas 缓存地形
- 只在地图移动/缩放时更新
- 分层渲染（地形、领土、城市、军队）

### 2. 领土渲染优化
**问题：** Voronoi 计算耗时
**解决：**
- 缓存领土边界
- 只在城市归属变化时重算
- 使用 Web Worker（可选）

### 3. 对象池
**问题：** 频繁创建/销毁对象
**解决：**
- 军队对象池
- 粒子对象池
- 减少 GC 压力

### 4. 事件系统
**问题：** 事件处理耦合严重
**解决：**
- 实现发布-订阅模式
- 解耦游戏逻辑
- 便于扩展
