# 指令名称：Step 1 - 时空数据库提取 (ETL Protocol v4.0)

### 1. 核心任务目标
作为**首席军事地理分析官**，你的任务是解析《资治通鉴》、《中国历代战争史》《战国军事史》等源码，提取特定年份的**原子级机动数据**。
你的输出必须能够让后期制作人员直接在 ESRI 地形图上通过坐标连线，复现真实的部队位移与战术博弈。

### 2. 五大“硬核”约束 (Hard Constraints)
1.  **【地理对撞与路径补全】**：严禁“瞬移”。若起点与终点间存在黄河、太行山、秦岭、大巴山等物理阻隔，必须在 `March_Nodes` 强制插入**渡口或隘口**的坐标。
2.  **【战术位移差逻辑】**：在 `Tactical_Logic` 中必须分析双方主力的空间相对位移。例如：赵军主力远在燕地（坐标A），秦军突袭邺城（坐标B），需算出此位移差导致的战略真空。
3.  **【兵种战术参数】**：严禁填 N/A。必须识别 `Unit_Type`（如：秦锐士、胡服骑射），并根据地形标注其适性（如：山地限制车骑，强弩封锁狭道）。
4.  **【补给生命线】**：标注 `Logistics_Line`（补给线）。识别该行动的粮草支撑点（如：敖仓、石门）及运输媒介（如：河运、栈道）。
5.  **【兵力强制推算】**：史料为主，没有再推算：大将/主力=100k (Est)，偏师/守军=30-50k (Est)，必须标注 (Est) 字样。

### 3. 数据输出标准格式 (Schema)

请生成 Markdown 表格，包含以下列：

| 字段名 | 说明 |
| :--- | :--- |
| **Seq_ID** | 序号 (01, 02...) |
| **Subject** | 将领或军团名 (如: 王翦军团) |
| **Faction** | 势力ID+颜色 (参照下方列表) |
| **Unit_Type** | 兵种特质 (如: 秦锐士/12石强弩) |
| **Action** | 精细化战术动作 (如: 侧翼迂回/奇袭) |
| **Origin -> Target** | 起点 -> 途经点(含渡口) -> 终点 (必须带 [Lat, Lng]) |
| **Logistics_Line** | 补给支点 (如: 上郡-太原转输) |
| **Tactical_Logic** | 战术成败因果 (如: 利用赵军主力伐燕之“空间差”) |
| **Terrain_Physics** | 地形物理约束 (如: 太行锁钥：两崖壁立，仅容一轨) |
| **Source_Quote** | 源码引用 |
