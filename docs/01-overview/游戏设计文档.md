## 1. 游戏概述

《战国地图战争》是一款基于网页的战略模拟游戏，玩家可以体验中国战国时期的战争与外交。游戏采用JavaScript开发，使用HTML5 Canvas进行地图渲染。

## 2. 核心机制

### 2.1 时间系统
- 游戏时间以季节为单位推进
- 每个季节持续15秒（15000毫秒）
- 每年包含4个季节（春夏秋冬）

### 2.2 资源系统
- **兵力**：每个城市有固定兵力，每年增长
  - 初始兵力：10,000
  - 年增长：1,000
  - 最大兵力：10,000

### 2.3 势力兵力系统（2025-10-31更新）
- **势力总兵力**：势力的总兵力 = 该势力所有城市兵力的总和
- **战争兵力消耗**：
  - 触发战争事件时，不再凭空生成兵力
  - 从势力的总兵力中按比例扣除所需兵力
  - 各城市按其兵力占比平摊扣除
- **兵力不足处理**：
  - 如果势力总兵力不足事件所需兵力，自动补全到所需数量
  - 将该势力所有城市兵力清零
  - 仍然生成所需规模的军队（保证历史事件正常进行）

## 3. 数据结构

### 3.1 势力(Faction)
```javascript
{
    id: string,      // 唯一标识符（如"qin", "baiyue"）
    name: string,    // 显示名称（如"秦国", "百越"）
    color: string    // 十六进制颜色代码
}
```

### 3.2 城市(City)
```javascript
{
    id: string,          // 唯一标识符（如"xianyang", "guangzhou"）
    name: string,        // 城市名称（如"咸阳", "广州"）
    factionId: string,   // 所属势力ID
    latitude: number,    // 纬度
    longitude: number,   // 经度
    importance: string,  // 城市等级（"major" 或 "minor"）
    troops: number       // 当前兵力
}
```

### 3.3 军队(Army)
```javascript
{
    id: string,          // 军队唯一ID
    factionId: string,   // 所属势力ID
    x: number,          // 地图X坐标
    y: number,          // 地图Y坐标
    targetX: number,    // 目标X坐标
    targetY: number,    // 目标Y坐标
    troops: number,     // 军队规模
    speed: number,      // 移动速度
    path: Array,        // 移动路径
    isMoving: boolean,  // 是否在移动中
    startTime: number   // 移动开始时间
}
```

## 4. 历史事件系统

### 4.1 事件结构
```javascript
{
    id: string,         // 事件ID
    year: number,       // 发生年份（公元前为负数）
    season: number,     // 季节（0=春,1=夏,2=秋,3=冬）
    title: string,      // 事件标题
    description: string,// 事件描述
    actions: [          // 事件触发的游戏行为
        // 可能的行动类型见下文
    ]
}
```

### 4.2 行动类型

#### 4.2.1 生成军队 (spawn_army)
```javascript
{
    type: "spawn_army",
    factionId: string,      // 军队所属势力
    autoSelectNearest: boolean, // 是否自动选择最近城市
    toCityId: string,       // 目标城市ID
    troops: number,         // 军队规模（从势力总兵力中扣除）
    speed: number           // 移动速度
}
```

**重要说明**：生成军队时会自动从该势力的所有城市中按比例扣除兵力，而不是凭空生成。

#### 4.2.2 围城 (siege)
```javascript
{
    type: "siege",
    attackerFactionId: string,  // 攻击方势力
    targetCityId: string,      // 目标城市
    duration: number,          // 围城持续时间(毫秒)
    casualtyRatio: {           // 伤亡比例
        defender: number,      // 防守方伤亡比例
        attacker: [number, number] // 攻击方伤亡比例范围
    }
}
```

#### 4.2.3 城市易主 (transfer_city)
```javascript
{
    type: "transfer_city",
    cityId: string,        // 城市ID
    fromFactionId: string, // 原属势力
    toFactionId: string    // 新属势力
}
```

## 9. 核心函数说明

### 9.1 兵力管理函数（2025-10-31新增）

#### getFactionTotalTroops(factionId)
计算指定势力的总兵力
- **参数**：factionId - 势力ID
- **返回**：该势力所有城市兵力的总和
- **用途**：查询势力当前可用兵力

#### deductTroopsFromFaction(factionId, requiredTroops)
从势力的城市中扣除兵力
- **参数**：
  - factionId - 势力ID
  - requiredTroops - 需要扣除的兵力数量
- **返回**：实际可用的兵力数量
- **逻辑**：
  1. 计算势力总兵力
  2. 如果总兵力不足，清空所有城市兵力并返回所需数量（自动补全）
  3. 如果总兵力充足，按各城市兵力占比平摊扣除
  4. 处理四舍五入导致的剩余，从兵力最多的城市继续扣除
