# 《历史大乱斗》直播观赏性分析报告

> **呈送对象**：游戏策划师  
> **项目名称**：历史大乱斗（MAPWAR）  
> **当前版本**：2026-06-30  
> **撰写目的**：分析游戏AI自动游玩在直播场景下的观赏性，记录已完成的优化，明确下一步发展方向。

---

## 一、项目基本面

### 1.1 产品定位

> **「不出声解说也能看懂」的历史大乱斗观察赛。**

AI自动推演的历史战略沙盒。玩家几乎不操作——选阵容、调速暂停、跟拍军团、下达远征指令。707个势力在真实亚洲大地图上自己打，玩家像上帝/史官在旁边看。

**核心判断标准**：「好不好看、有没有悬念」，不是传统4X的「好不好玩」。

### 1.2 数据规模

| 指标 | 数值 |
|------|------|
| 总据点 | **707** |
| 大城 | 20 | 中城 | 60 | 小城 | 484 | 关隘 | 143 |
| 势力总数 | **707**（每城一势力） |
| 专属武将 | 707 | 专属精锐 | 707 |
| 文化区 | 14 区 |
| 精锐 T0 | 21 | T1 | 126 | T2 | 313 | T3 | 247 |

### 1.3 地理覆盖

```
中原(80)  草原(79)  岭南(66)  青藏(66)  江南(61)
川蜀(49)  中亚(49)  东北(45)  北方(44)  日本(38)
滇缅(36)  河西(35)  西域(35)  朝鲜(24)
```

707座城均以**真实历史坐标**定位，每城配**1900年以前可考的史地名**。

## 二、已有优势

### 2.1 地图大、真实
每一座城都是可查证的史地坐标。观众能看到自己家乡的古称（「汴梁」不是「开封」），认出历史故事的发生地（长平、赤壁、潼关）。地图本身就是内容。

### 2.2 旗号系统
707个势力各有1-2汉字旗号（「秦」「唐」「德川」「蒙古」），深旗白字/浅旗黑字自动适配。无声观众靠旗号就能认人。

### 2.3 名将+精锐
707名武将 × 707支精锐（T0-T3）。战场上出现「白起率秦之锐士 vs 廉颇率赵边骑」。名场面制造机。

### 2.4 镜头系统
跟拍模式（锁定一支军团）、导播模式（自动跳热点）、自由模式。

### 2.5 军情播报
六类军情（灭/歼/复/征/隘/占），句式统一。天生的弹幕触发点。

### 2.6 反雪球机制
`枪打出头鸟`：≥40城触发，20%概率合击。推演验证终局存活34-81势力，无领先者破百城。

### 2.7 文化区差异
14区分五级攻防（草原×1.2野/蜀地×1.2守），不同文化打得不一样。

## 三、关键问题

### 3.1 🩸 跟拍军团存活率低（核心瓶颈）

**现状**：跟拍军团打2-3场仗就濒临解散。

**根因**：
- 战后恢复仅30%。一场硬仗10000兵打到3000 → 恢复后仅5100
- 解散阈值5000，残血直接回家
- 非跟拍军团无任何补兵手段
- 中后期大量小城（2000-20000驻军），即使50000+大军团连打5-6城也必然消耗殆尽
- 远征需打20+城才能抵达目标，按当前消耗模型几乎不可能

**直播后果**：观众刚认识一支军团（记住它的武将、精锐），它就没了。主角切换频率过高，观众无法建立情感羁绊。

### 3.2 缺乏叙事弧线
纯沙盒推演，无「开播—高潮—结局」结构。

### 3.3 信息密度不均
707城同时运转，重要事件和日常摩擦视觉权重差不多。

### 3.4 缺少赛后总结
一局打完无编年史/统计面板。复盘只能凭记忆。

## 四、已完成优化

### ✅ 战斗对峙条动量拉锯（2026-06-30）

**问题**：战斗"开局即定胜负"——弱方从第一秒匀速滑向零，毫无悬念。

**方案**：OU过程噪声 + 三阶段相位包络，不改胜负锁定。

| 阶段 | 时长 | 效果 |
|------|------|------|
| 开局渐入 | 33% | 噪声从0平滑升到满幅 |
| 中段拉锯 | 34% | 满幅波动，对峙条来回晃 |
| 末段收敛 | 33% | 噪声衰减+地板兜底，确保弱方归零 |

**效果**：观众看到对峙条在中段来回晃、弱方偶尔「反推」回来，末段果断收束到赢家。战斗结果完全不变。

**修改文件**：`src/combat/BattleField.ts`（新增 `momentumValue/Target/Timer` 字段 + `getPhaseEnvelope()` + `updateMomentum()` 方法，重写 `update()` 伤害段）

## 五、下一步方向：士气/恐惧系统

### 5.1 设计目标

让跟拍军团凭借「名将+精锐」的质量优势，在连续攻城战中减少消耗、维持兵力。核心思路：**精锐之名，可溃敌胆**。

### 5.2 九级恐惧等级

双方各计算一个质量分，取分值差决定恐惧效果。

**加算公式**：

```
将领分：名将=3, 普将=1, 无将=0
精锐分：T0=4, T1=3, T2=2, T3=1, 无= -1
总分   = 将领分 + 精锐分    （范围 -1 ~ 7，恰好 9 档）
```

| Lv | 总分 | 组成举例 |
|:--:|:----:|----------|
| **9** | 7 | 名将 + T0（如白起 + 秦之锐士） |
| **8** | 6 | 名将 + T1 |
| **7** | 5 | 名将 + T2 |
| **6** | 4 | 名将 + T3 / 普将 + T0 |
| **5** | 3 | 普将 + T1 / 名将 + 无精 |
| **4** | 2 | 普将 + T2 |
| **3** | 1 | 普将 + T3 / 无将 + T0 |
| **2** | 0 | 无将 + T1 / 普将 + 无精 |
| **1** | -1 | 无将 + T2/T3/无 → 无恐惧 |

### 5.3 抵消规则

双方各算Lv，**仅当己方 > 敌方时恐惧生效**。同等Lv完全抵消（名将对名将+T0对T0 = 零恐惧 = 公平对决）。

| 优势 | 恐惧效果（弱方掉血加速） |
|:--:|------|
| +1 | ×1.10 |
| +2 | ×1.20 |
| +3 | ×1.30 |
| +4 | ×1.40 |
| +5+ | ×1.50 |

### 5.4 实战推演

> **白起（名将）+ 秦之锐士（T0）= Lv9，攻邯郸**
> 邯郸守将廉颇（名将）+ 赵边骑（T2）= Lv7
> diff = 2 → 恐惧 ×1.20
> 廉颇的赵边骑在恐惧下掉血快20%，白起的秦之锐士战损更少。

> **白起（Lv9）攻无名小城**
> 守军无将无精 = Lv1
> diff = 8 → 恐惧 ×1.50
> 摧枯拉朽，秦之锐士几乎无损。

> **白起（Lv9）vs 项羽（名将+T0=Lv9）**
> diff = 0 → 零恐惧
> 同为顶级，公平死战。这才是观众想看的「名将巅峰对决」。

### 5.5 技术实现

恐惧效果作用在 `BattleField.update()` 的伤害计算阶段——己方Lv > 敌方Lv时，弱方DPS乘以恐惧系数。不改胜负判定逻辑，只加速弱方掉血。

**需要新增的数据**：战斗双方各查一次 `GeneralSkills.getProfile()` + `ExpeditionLegions.getTier()` 即可得出Lv，无需新增字段。

## 附录：设计原则

1. **好看 > 好玩**。每一个设计决策都要问：「对着直播镜头，这东西好看吗？」

2. **历史是护城河**。707个真实据点不可复制。所有改进都强化这个优势，不稀释它。

3. **观众需要「锚」**。跟拍军团就是主角。让主角活久一点，让它带着观众走完一段史诗。

4. **节奏 > 细节**。80%的屏幕时间给20%的关键剧情。边境摩擦不值一提，长安易主才是大事。

5. **编年史 = 内容资产**。一局打完的战后总结本身就是可传播的内容。
