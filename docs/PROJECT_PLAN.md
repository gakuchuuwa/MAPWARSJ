# MAPWAR 项目规划书

> **产品定性**：**历史大乱斗**——在真实历史地理地图上，百余势力（历史上的政权、民族、大家族）各据一城，展开跨越时空的沙盘混战。  
> **核心规则**：**一个势力 = 一个城**。每个 `factionId` 对应且仅对应一个城市。势力来源于三个大类：政权、民族、大家族。  
> **不是**：Civ6 式 4X、全战式回合、星际式微操 RTS、世界模拟器。

## 1. 游戏定位

### 1.1 一句话

**"历史大乱斗"**——在真实历史地理地图（全球经纬度 2D 地图）上，上百个来自中国历史与周边历史的势力（政权、民族、大家族），各据一城，展开跨越时空的沙盘混战。时间实时流逝（可暂停/调速），军队在路上真实移动，接战持续结算。

### 1.2 核心设计原则

1. **一势力一城**：每个 `factionId` 在数据层对应且仅对应一个城市。这是游戏的根基规则。
2. **势力三大来源**：
   - **政权**：历史上的王朝、王国、割据政权（如秦、汉、唐、宋、明、匈奴、突厥、蒙古等）
   - **民族**：历史上的民族/部族实体（如羌、氐、鲜卑、契丹、女真、百越等）
   - **大家族**：历史上的士族门阀、地方豪强（如琅琊王氏、兰陵萧氏、清河崔氏等）
3. **真实历史地理**：城市坐标基于真实经纬度；50km 间距冲突时优先删除名气较小的据点，两难时可 ±1–2 km 微调，严禁胡编大挪移。
4. **严肃历史沙盘**：所有城市/势力均基于真实历史存在；不做随机生成大陆、不做虚构文明。

### 1.3 时间规则

| 概念 | 本作 | 不做 |
|------|------|------|
| 时间 | **实时流逝**，随时 **暂停 / 1× / 2× / 4×** | 回合制、结束回合、等待全员点下一步 |
| 战斗 | 接触后 **持续进行**（DPS），可暂停观看 | 打一回合停一下、进独立回合战场 |
| 参考 | 历史模拟 + 可暂停的实时节奏（类似带暂停的单机 RTS） | 《全面战争》战役回合、任何「战略回合」 |

### 1.4 类型与平台

| 维度 | 定义 |
|------|------|
| 类型 | **历史大乱斗**（Historical Grand Melee）—— 历史沙盘 + 实时战争 |
| 画面 | **2D** 战略地图 + 单位/领土叠加层 |
| 平台 | Web 浏览器，PC 为主 |

### 1.5 明确不做（范围边界）

- **不做回合制**（含战略回合、战斗回合、「下一回合」按钮）。
- **不做 Civ6 式 4X**（无科技树、无时代升级、无开拓者铺城等）。
- **不做虚构文明/随机地图**。
- 不做 3D 战场、不做竞技微操 RTS。
- 不以伪直播、全球指令同步为当前目标。

### 1.6 与旧设计的区别

| 旧设计（2026-05-24 前） | 新设计（2026-05-24 起） |
|-------------------------|-------------------------|
| 实时可暂停的 2D《文明 6》4X | **历史大乱斗** |
| 沙盒模式为主 | **势力混战为核心** |
| 时代从原始到未来 | **固定历史背景**（不跨时代） |
| 少数文明（4~8 个模板） | **百余势力**（政权/民族/大家族） |
| 一势力多城 | **一势力一城** |
| 历史剧本包为沙盒外扩展 | 历史战役脚本可并行存在 |
| 全球地图 + 全球文明 | **聚焦东亚/中国历史**，引擎层支持全球 |

## 2. 扮演层：玩家扮演什么

### 2.1 默认模式 — 历史大乱斗

玩家以 **观察者/导演** 身份进入，观看或接管任意势力，在历史大乱斗中：

| 玩家可做的事 | 说明 |
|-------------|------|
| 接管势力 | 选择一个势力，控制其军队与城市 |
| 调兵遣将 | 征募军团，沿路网实时机动，发起攻城/野战 |
| 扩张领土 | 占领他方城市，势力面自动推进 |
| 观看历史 | 历史事件脚本自动播放，展示跨越数百年的战役 |
| 导演模式 | 暂停时间，查看战局，接管关键节点 |

### 2.2 历史大乱斗 vs 历史剧本

| 模式 | 地位 | 说明 |
|------|------|------|
| **历史大乱斗** | **主模式** | 所有势力同时存在于地图上，自由混战；时间可流动或步进 |
| **历史战役剧本** | **附属内容** | 按年代编排的战役序列（约 958 场），可在乱斗模式下播放，也可独立观看 |

## 3. 世界观与内容范围

### 3.1 势力体系

势力分为三大类，共约 200+ 个（数据见 `src/data/factions.ts`）：

| 大类 | 示例 | 说明 |
|------|------|------|
| **政权** | 秦、汉、蜀、吴、晋、隋、唐、宋、辽、金、元、明、清、匈奴、突厥、蒙古 | 历史上存在过的王朝/王国/游牧汗国 |
| **民族** | 羌、氐、鲜卑、契丹、女真、百越、蛮、獠、室韦、靺鞨 | 历史上的民族/部族实体 |
| **大家族** | 琅琊王氏、兰陵萧氏、清河崔氏、范阳卢氏、太原王氏 | 历史上的士族门阀/地方豪强 |

### 3.2 城市体系

- 每个势力对应 **一个城市**（`one-city-per-faction`）
- 城市数据包含：经纬度、类型（大城/中城/小城/关隘/渡口/战场）、兵力、所属势力
- 严格遵守 **50km 最小间距规则**
- 真实地理坐标；过近优先删小留大，必要时可 **±1–2 km 微调** 凑间距，**严禁**胡编大挪移

### 3.3 地图范围

| 维度 | 范围 |
|------|------|
| **地理** | **真实地球** 2D 地图（全球经纬度）；内容聚焦中国及周边（东亚/中亚/东北亚） |
| **引擎能力** | 全球任何坐标均可显示（Leaflet + 高程瓦片）；玩法数据当前主要覆盖中国区 |
| **内容扩展** | 未来可添加全球其他历史区域（欧洲、中东等） |

## 4. 核心循环（Core Loop）

```
历史大乱斗核心循环：
  预览全局 → 选择势力 → 调兵/建设 → 征战/防守 → 吞并/被吞
      ↑__________________________________________|
                （全程：世界时钟运行，可暂停/调速）
```

| 阶段 | 说明 | 系统支撑 |
|------|------|----------|
| 预览全局 | 查看地图上的所有势力分布、兵力对比 | 地图渲染 + 势力色块 |
| 选择势力 | 选择一个已有势力接管（或保持导演模式） | UI 系统 |
| 调兵/建设 | 在城市征募军团，沿路网机动；升级城防/产出 | `SiegeManager`、城市字段 |
| 征战/防守 | 实时行军，接触即战；攻城/野战持续结算 | `BattleField`、`CombatSystem` |
| 吞并/被吞 | 胜则占领城市，败则丢失城市；势力可被完全消灭 | `SiegeManager.resolveChain` |

## 5. 核心系统设计

### 5.1 地图与呈现（2D）

> 地图表面管线已打通——全球真实经纬度上的 2D 地形可视化已完成。

**核心入口**：`src/map/GameMap.ts`（Leaflet `L.Map`，zoom **4–13**，默认中心长安 `34.26°N, 108.94°E`）。

| 组件 | 文件 | 作用 |
|------|------|------|
| 程序化地形 | `HillshadeLayer.ts` + `HillshadeWorker.ts` | 拉取全球 Terrarium 高程瓦片，山体阴影 + 海拔分层着色 |
| 备用底图 | `TileMapConfig.SOURCES.ESRI_SHADED` | ArcGIS 立体地形瓦片，可切换 |
| 河流（栅格） | `RiverOverlayLayer.ts` + `RiverWorker.ts` | 水域叠加 |
| 河流（矢量） | `VectorRiverLayer.ts` | Natural Earth 矢量河流，zoom 9–12 |
| 长城 | `VectorWallLayer.ts` | 矢量长城，zoom 9 |
| 战略六边格 | `StrategicGridLayer` + `GridSystem` | 六边格锚定在经纬度上 |
| 领土涂色 | `TerritorySystem` + Worker | 城市 BFS 六边着色 |
| 道路 | `SimpleVectorRoadRenderer` | 史实矢量路 + 寻路图 |
| 城市/单位/特效 | `CityManager`、`GlobalUnitRenderer`（Canvas） | 玩法层 |

### 5.2 点：据点（Cities）

据点类型（`CityType`）：`big_city` / `medium_city` / `small_city` / `pass` / `ferry` / `battlefield`。

| 机制 | 设计 | 现状 |
|------|------|------|
| 补给中心 | 大城产粮；关隘/小城为粮道节点 | 字段 `supply` 已有，玩法未闭环 |
| 城防 | `defense` 影响攻坚阶段 | 攻城逻辑部分在 `SiegeManager` |
| 兵源 | `population` / `maxTroops` | 数据字段已有 |
| 时间可见 | `startYear` / `endYear` | 已实现（`RegionSystem`） |

### 5.3 线：行军与寻路

- **阶段 A（当前）**：城市/路口为节点，矢量道路为边；军团沿路网移动。
- **阶段 B（后期）**：全图六边/栅格化，地形阻力寻路。

**现状**：`RoadRegistry` + `HexPathFinder` fallback；`TerrainSpeedSystem` 已采样地形速度。

### 5.4 面：势力范围（ZOC）

- 占点即时更新势力色块（六边格 BFS 扩张）。
- 进入敌方 ZOC：视野暴露、移速 −30%、粮草 ×2；断粮则持续掉士气与兵力。

**现状**：`TerritorySystem` 从城市 BFS 涂色；ZOC 惩罚与断粮规则未完全实装。

### 5.5 战斗系统

统一底层 **`BattleField` 区域战引擎**；野战与攻城规则分轨、代码合流。

| 类型 | 核心体验 | 胜利条件 |
|------|----------|----------|
| **野战** | 机动、包围、夹击 | 击溃士气或歼灭 |
| **攻城** | 坚守、攻坚、围点打援 | 占领城市核心 |

**无缝战斗**：接战判定为地图距离/邻接/区域重叠，不加载独立场景。

### 5.6 历史事件引擎

沙盒本体 + 外部脚本按条件注入事件。

**事件类型**（`types/core.ts`）：

| 类型 | 用途 | 数据要点 |
|------|------|----------|
| `siege` | 攻城 | `attackerFactionId`、`defenderCityId`；可改归属 |
| `field_battle` | 野战 | 双方势力、`location`、来源城；不改城市归属 |
| `narrative` | 叙事 | `year`、`title`、`description`；可无战斗 |

### 5.7 资源与经济（首版简化）

- 首版：`兵力` + `粮食` 即可支撑征兵与出征。
- 后期再加：金钱、装备、士气等。

### 5.8 AI

- **战略 AI**：`AIController` + 行为树（默认关闭）。
- **历史剧本 AI**：事件脚本即「导演」；优先保证战役可稳定播放。

## 6. 时间与战斗：实时规则

### 6.1 战略时间（非回合）

| 规则 | 说明 |
|------|------|
| 世界时钟 | 持续前进（年/季可映射为游戏内日历）；无「结束回合」 |
| 玩家控制 | 暂停、恢复、多档 `timeScale`（已有 `TimeSystem.timeScale`） |
| 行动排队 | 下令瞬间生效或进入队列；下令不消耗「回合」 |
| AI | 与玩家共用同一时间流；暂停时默认冻结 AI |

### 6.2 战斗结算（即时 DPS）

接战后，双方在地图上持续结算伤害，直至一方溃败、撤退或占点成功。

```
每秒伤害 = f(兵力, 基础攻击, 武将统率, 地形克制, 兵种克制) − 敌方防御（及城防）
```

兵种克制与地形修正见 `src/config/GameConfig.ts`。

| 间距规则 | 50km / 0.4° 最小间距 | 严格执行；冲突时删除名气较小者 |

### 8.3 战役脚本

| 阶段文件 | 目标场次 | 录入状态（2026-05） |
|----------|----------|---------------------|
| `index_01_qin.ts` | ~45 | 已有 |
| `index_02_han.ts` | ~92 | 已有 |

### 8.4 数据红线

1. 禁止未注册的 `cityId` / `factionId`。
2. 一个 `factionId` 应只对应 **一个城市**（当前数据中部分势力拥有多城，需逐步清理）。
3. `siege` 必须攻击方 + 防守城；`field_battle` 必须双方来源与战场坐标。
4. 城市间距 ≥ 50km（经纬度差 ≥ 0.4°）。
5. 间距冲突优先删小留大；两难时可 ±1–2 km 微调，**严禁**胡编大挪移。
