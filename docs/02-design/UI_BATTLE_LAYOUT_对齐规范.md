# 历史大乱斗 — 战斗 UI 对齐与排版规范

> **必读**：本文档旨在指导所有 AI 在修改《历史大乱斗》（MAPWARSJ）战斗面板（`CombatUI.ts`）时，必须严格遵守的布局与对齐原则。
> 严禁任何 AI 在未阅读并深刻理解本文档的情况下，随意更改 `flex-direction`、`margin: auto` 或 `text-align` 等布局属性。

---

## 一、核心排版总则：【文字锚定边缘，数据聚焦中央（两极分化对齐）】

在《历史大乱斗》的区域冲突（CombatUI）大面板中，绝对禁止使用纯碎的“向中靠拢”（导致起步边缘锯齿化、阅读破裂）或纯粹的“向外对齐”（导致中央产生巨大的视觉黑洞）。

唯一且绝对的最优排版法则是：**分离式两极对齐法（Split Alignment）**。

*   **文字墙（势力名、军团名）**：必须死死贴在屏幕的**最外侧边缘（靠近武将画像）**。提供稳定的垂直扫描锚点，杜绝参差不齐的锯齿边缘。
*   **数据流（兵力、倍率链、徽章）**：必须被推向屏幕的**正中央（交叉双剑附近）**。确保玩家在视线锁定中央战场时，能用余光直接看清核心数据。

---

## 二、底层实现原理（Flexbox 弹性推离）

为了实现文字靠外、数据靠内的两极分化效果，我们在 `CombatUI.ts` 中巧妙地运用了 Flexbox 布局下的 `margin: auto` 特性，而不是死板的 `justify-content`。

### 1. 进攻方（Attacker / 左侧）
进攻方全局使用 `flex-direction: row`（从左向右排列）。
*   **DOM 顺序**：`[名字 Span]` -> `[数据 Span]`。
*   **推离魔法**：在左侧的 `[数据 Span]` 上设置 `margin-left: auto;`。
*   **渲染结果**：名字停留在最左侧边缘，`margin-left: auto` 会自动吃掉中间所有多余的空白空间，把后面的数据狠狠推向最右侧（贴近中央双剑）。
*   **最终视觉**：`[泰国] -------------------- [1.2×1.2×1.2] (双剑)`

### 2. 防守方（Defender / 右侧）
防守方全局使用 `flex-direction: row-reverse`（从右向左反向排列），以达成绝对的对称镜像。
*   **DOM 顺序**：`[名字 Span]` -> `[数据 Span]`。
*   **推离魔法**：在右侧的 `[数据 Span]` 上设置 `margin-right: auto;`。
*   **渲染结果**：因为是反向渲染，最先 Append 的名字会停留在最右侧边缘。此时 `margin-right: auto` 会把后面的数据狠狠推向最左侧（贴近中央双剑）。
*   **最终视觉**：`(双剑) [1.2×1] -------------------- [陇州]`

## 三、为什么这是终极方案？（防坑指南）

过去曾经发生过因 AI 在“纯两边对齐”和“纯中间对齐”之间反复横跳，导致 UI 灾难的历史教训：

1.  **纯中间对齐的灾难**：为了让数据靠近中央，把整个 block 向中间推。结果导致左侧长短不一的名字（如“泰国”和“秦之锐士 军团”）的起始字完全错位，形成严重的“锯齿边缘”，玩家阅读时眼球需要不断左右寻找起点，极其痛苦。并且武将画像旁边空无一物，产生剥离感。
2.  **纯两边对齐的灾难**：为了让名字整齐，把整个 block 向两边推。结果导致最核心的动态战报数据（如 `1.82万` 兵力）距离中央的对峙槽和双剑十万八千里。整个屏幕中央形同“黑洞”，数据毫无战场交锋的压迫感。

**分离式两极对齐（Split Alignment）**完美吸取了双方的优点，丢掉了所有的缺点：名字提供静态的稳重结构，数字提供动态的交锋张力。此规范锁定后，任何人不得擅自更改！
