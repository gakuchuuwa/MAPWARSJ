# 历史大乱斗 — 武将技（General Skills）系统设计

> **必读**：本文档定义武将技的分类、触发条件、数值边界和实现挂载点。新增技能必须在此框架内扩展。

---

## 〇、游戏特性约束（设计前提）

| 约束 | 说明 |
|:---|:---|
| **实体模型** | 一据点 = 一势力 = 一军团 = 一将领。军团分三类：据点军团、剧本军团（scripted campaign）、远征军团（expedition）。 |
| **兵种** | 仅骑兵、步骑混编（无纯步兵，无兵种克制链）。 |
| **地形** | 山地、平原、水域（无森林/沙漠等细分）。 |
| **数值维度** | 只有兵力（troops）。无将领五维、无士气、无内政。 |
| **战斗模型** | `Battle.calculateBattleParameters()` 算战力 → 逐帧扣血 → `resolve()` 分级恢复。已有战后恢复系统（攻城 `10%–40%`，野战 `50%`）。 |
| **核心痛点** | 剧本军团连续远征时，打一两场就残了——武将技的目标是**延长连续作战能力**，而非让单场战斗更花哨。 |

---

## 一、武将技四大分类（功能分类，非稀有度分类）

摒弃「神将/逆境/场景/兵种」的稀有度分档法，改为按**功能角色**分类。每类回答一个不同的问题：

| 分类 | 回答的问题 | 触发时机 |
|:---|:---|:---|
| **常驻战力** | 这个将领本身有多能打？ | 全程在线 |
| **逆境爆发** | 打不过的时候能不能翻？ | 战斗中满足条件 |
| **战场特化** | 在什么场景下特别强？ | 根据敌方类型/己方角色 |
| **战后续航** | 打完了能恢复多少？ | 战斗胜利结算时 |

---

## 二、常驻战力（Identity）

> **无条件、永久在线。这个将领就是比别人硬。**

底层机制：**战力倍率**（贯穿整场战斗，总战力 × 固定倍率）。

| 技能名 | 效果 | UI 表现 | 代表将领 |
|:---|:---|:---|:---|
| **【军神】** | 常驻战力 `×1.2` | 金色大字「军神」+ 立绘光晕 | 白起、韩信、李靖 |
| **【霸王】** | 常驻战力 `×1.2` | 暗金大字「霸王」+ 霸气震屏 | 项羽 |
| **【飞将】** | 常驻战力 `×1.2` | 银色「飞将」+ 疾风特效 | 李广、吕布 |

> **设计说明**：军神/霸王/飞将在底层完全等效（都是 `×1.2`），区别仅在 UI 表现——不同字色/特效区分将领气质。**同一将领只选其一，三个技能互斥。**

### 挂载点

在 `Battle.calculateBattleParameters()` 中，调用 `rollSideEffectivePower()` **之前**，对拥有常驻技能的将领的 `IBattleUnit` 注入战力倍率。这是构造时一次性计算，战斗中不再变动。

---

## 三、逆境爆发（Comeback）

> **逆风或残血时触发，提供翻盘窗口。直击"打残了还能继续打"的痛点。**

底层机制：**战力倍率 / 瞬间恢复 / 伤害免疫**。

### 3.1 逆风触发（己方兵力 < 敌方兵力）

判定时机：**文化修正后、随机掷点前**（确定性判定，不依赖 `×[0.8,1.2]` 掷点）。

| 技能名 | 触发条件 | 底层机制 | 效果 | UI 表现 | 代表将领 |
|:---|:---|:---|:---|:---|:---|
| **【背水】** | 己方兵力 < 敌方兵力 | 战力倍率 | 战力 `×1.2` | 爆衣特效 + 蓝色「背水」大字 | 韩信、徐晃 |
| **【破釜】** | 己方兵力 < 敌方兵力 | 战力倍率 | 战力 `×1.2`（同背水） | 沉船特效 + 暗红「破釜」大字 | 项羽及项家军部将 |
| **【威压】** | 己方兵力 < 敌方兵力 | 伤害免疫 | 3 秒无敌 | 敌方头像眩晕 + 紫色「威压」大字 | 吕布、冉闵等顶级猛将 |
| **【神算】** | 己方兵力 < 敌方兵力 | 瞬间爆伤 | 开战瞬间扣敌方当前兵力 `15%` | 八卦阵/火攻特效 + 「神算」大字 | 诸葛亮、张良 |

> **设计说明**：【背水】和【破釜】底层完全等效，互斥——项羽系用【破釜】，其余将领用【背水】。【神算】在开战第 1 帧触发，只触发一次。

### 3.2 残血触发（自身兵力 < 初始的 N%）

| 技能名 | 触发条件 | 底层机制 | 效果 | UI 表现 | 代表将领 |
|:---|:---|:---|:---|:---|:---|
| **【不屈】** | 自身兵力 < 初始的 `20%` | 瞬间恢复 | 恢复初始兵力的 `20%`（不超过 `maxTroops`） | 红光护体 + 「不屈」大字 | 周泰、典韦 |
| **【坑杀】** | 敌方兵力 < 其初始的 `30%` | 瞬间爆伤 | 扣除敌方当前兵力的 `70%` | 猩红震屏 + 「坑杀」大字 | 白起（专属） |

> **设计说明**：【坑杀】不扣 100%——留 30% 给对手一丝生机，防止白起势力碾压所有 AI。每次战斗最多触发 1 次。

### 挂载点

- **逆风判定**：在 `Battle.calculateBattleParameters()` 中，掷点前判定。判定结果存入 `Battle` 实例。
- **战力倍率类**（背水/破釜）：在 `Battle.update()` 每帧重新检查条件（有可能兵力变化后条件不再满足）。
- **伤害免疫类**（威压）：在 `Battle.update()` 的伤害应用阶段前置检查 `isInvincible`（此字段已存在）。
- **瞬间爆伤/恢复类**（神算/不屈/坑杀）：在 `Battle.update()` 中作为特殊分支插入。
- **冷却/次数限制**：非「常驻」类技能需带 `cooldownSeconds` 或 `maxTriggersPerBattle` 字段。不屈和坑杀 `maxTriggersPerBattle = 1`。威压 `cooldownSeconds = 30`。

## 四、战场特化（Specialist）

> **在特定场景下获得优势，其他时候是白板。体现"用武之地"。**

底层机制：**战力倍率 / 伤害免疫**。

### 4.1 攻城/守城（敌方类型或己方角色）

| 技能名 | 触发条件 | 底层机制 | 效果 | UI 表现 | 代表将领 |
|:---|:---|:---|:---|:---|:---|
| **【攻城】** | 敌方为城池（`unitType === 'city'`） | 战力倍率 | 战力 `×1.2` | 「攻城」大字 + 冲车/云梯特效 | 王翦、乐毅 |
| **【铁壁】** | 己方为城池防守方（`unitType === 'city'`） | 战力倍率 | 战力 `×1.2` | 「铁壁」大字 + 城墙护盾特效 | 李牧、廉颇、曹仁 |
| **【坚守】** | 己方为城池防守方 | 伤害免疫 | 3 秒无敌 | 「坚守」大字 + 金色护盾 | 郝昭 |

> **设计说明**：【攻城】和【铁壁】对撞时，双方各 `×1.2`，最终效果等价于均无技能（因为双方同时乘以相同倍率，战损比不变）——但 UI 仍然播放以示博弈。【坚守】不与【威压】同时存在：一个将领最多一个免伤技。

### 4.2 讨夷（敌方势力类型）

| 技能名 | 触发条件 | 底层机制 | 效果 | UI 表现 | 代表将领 |
|:---|:---|:---|:---|:---|:---|
| **【讨夷】** | 敌方势力 `region` 为草原/西域/吐蕃/东北 | 战力倍率 | 战力 `×1.2` | 「讨夷」大字 + 骑兵突击特效 | 卫青、霍去病、冉闵 |

> **设计说明**：「游牧势力」在 MAPWAR 中定义为 `region` 属于 `STEPPE`、`CENTRAL_ASIA`、`TIBET`、`NORTHEAST` 四个文化区。判定时读取 `city.region` 字段。

### 挂载点

- 在 `Battle.calculateBattleParameters()` 构造时判定场景条件（敌方是否是城池、己方是否是守城方、敌方势力区域）。
- 场景条件在整场战斗中不变（攻城不会变成野战），所以场景特化是构造时一次性判定，无需每帧重算。

## 五、战后续航（Sustainability）

> **打赢之后恢复得比别人多。直接服务"连续远征"的核心痛点。**

底层机制：**瞬间恢复**（在 `resolve()` 结算阶段触发）。

| 技能名 | 触发条件 | 底层机制 | 效果 | UI 表现 | 代表将领 |
|:---|:---|:---|:---|:---|:---|
| **【连战】** | 战斗胜利 | 瞬间恢复 | 在系统分级恢复**之后**，额外恢复初始兵力的 `15%` | 「连战」大字 + 兵力条绿色回弹 | 高速推进将领 |
| **【劫掠】** | 战斗胜利 | 瞬间恢复 | 同上（初始兵力的 `15%`） | 「劫掠」大字 + 物资掠夺动画 | 游牧将领、董卓 |

> **设计说明**：【连战】和【劫掠】底层等效，互斥——游牧系用【劫掠】。【连战】/【劫掠】恢复与系统分级恢复**加算**。恢复总量 = 系统分级恢复 + 武将技恢复，但上限不超过初始兵力的 `80%`（防止"打赢等于满血"）。攻城胜利时两者都可触发——先系统恢复（10–40%），再加武将技恢复（15%），最终上限 80%。

### 挂载点

在 `Battle.resolve()` 中，系统分级恢复执行完毕后，检查胜利方将领是否拥有【连战】/【劫掠】，如有则追加恢复。恢复计算后由 `clamp` 到上限。

### 与现有恢复系统的数值关系

| 场景 | 系统恢复 | + 武将技恢复 | 理论上限 | 实际上限（80% cap） |
|:---|:---|:---|:---|:---|
| 攻大城 | 40% | +15% | 55% | 55%（未触上限） |
| 攻小城 | 20% | +15% | 35% | 35%（未触上限） |
| 野战 | 50% | +15% | 65% | 65%（未触上限） |
| 攻大城+战损极低 | 40% | +15% | 55% | 80% cap 不触发 |

> 当前数值下【连战】/【劫掠】不会触及 80% 上限，除非未来调高恢复率。上限作为安全阀保留。

## 六、底层实现机制（引擎积木）

无论规划多少技能，**所有实际效果最终归结为以下 4 种机制之一**。严禁为单个武将发明新机制。

### 1. 战力倍率（持续战力修正）
- **代码位置**：注入到 `rollSideEffectivePower()` 的输入参数中。
- **常驻类**：构造时注入，战斗全程不变。
- **逆境类**（背水/破釜）：`update()` 中每帧重判条件，动态开关。
- **场景类**（攻城/铁壁/讨夷）：构造时注入，战斗全程不变。

### 2. 瞬间爆伤（减少对方兵力）
- **代码位置**：`Battle.update()` 中作为特殊分支，在常规 DPS 扣血之外直接扣兵。
- 每场战斗每个技能最多触发 1 次（神算）或 1 次（坑杀）。

### 3. 瞬间恢复（增加己方兵力）
- **战斗中**（不屈）：`Battle.update()` 特殊分支。
- **战斗后**（连战/劫掠）：`Battle.resolve()` 追加阶段。

### 4. 伤害免疫（不掉血）
- **代码位置**：现有 `IBattleUnit.isInvincible` 字段。`Battle.update()` 和 `BattleField` 的伤害应用阶段前置检查此字段。
- 【威压】和【坚守】的本质区别仅在触发条件（逆风 vs 守城）和 UI 特效。

## 七、技能与将领的映射（数据层）

### 原则

- 技能绑定于**将领**，将领绑定于**军团**（据点/剧本/远征）。
- 同一将领可拥有多个技能，但需遵守互斥规则（见下文）。
- 技能数据独立于军团数据，便于历史将领精确配置。

### 互斥规则

| 技能对 | 规则 |
|:---|:---|
| 军神 / 霸王 / 飞将 | 互斥，同一将领只选其一 |
| 背水 / 破釜 | 互斥，同一将领只选其一 |
| 连战 / 劫掠 | 互斥，同一将领只选其一 |
| 威压 / 坚守 | 互斥，同一将领最多一个免伤技 |

### 叠加上限

- 战力倍率类技能（常驻 + 逆境 + 场景）**最多同时生效 2 层**（即最大 `1.2 × 1.2 = 1.44`）。
- 若同时满足 3 个战力倍率条件（如：军神 + 背水 + 攻城），只取优先级最高的 2 层。优先级：**常驻 > 场景 > 逆境**。

### 数据文件规划

仿照 `*ExpeditionLegions.ts` 模式，新建：

```
src/data/GeneralSkills.ts
```

结构：

```typescript
// 将领 → 技能列表映射
// key: 将领 ID（与 IBattleUnit.generalId 对应）
export const GENERAL_SKILLS: Record<string, string[]> = {
  baiqi: ['junshen', 'kengsha'],        // 白起：军神 + 坑杀
  hanxin: ['junshen', 'beishui'],        // 韩信：军神 + 背水
  xiangyu: ['bawang', 'pofu'],           // 项羽：霸王 + 破釜
  // ...
};
```

> `generalId` 与 `factionId` 不同——一个势力（如 `qin`）可以有多个将领（如 `baiqi`、`wangjian`）。

## 九、开发纪律

1. **最小侵入**：所有技能必须复用上述 4 大底层机制，作为现有战斗管线的增量插件。严禁为单个武将改 `Battle` 核心循环。
2. **先挂载点，后技能**：每新增一个触发条件，先在 `Battle` 中预留挂载点（如 `onConditionMet` 回调），再在挂载点上挂技能效果。
3. **重 UI 轻数值**：战力倍率严格限制在 `×1.2`。技能力量的 60% 来自 `CombatUI.ts` 的**光效、大字、震屏、立绘动效**——没有特效的技能没有灵魂。
4. **判定必须确定**：所有触发条件在相同输入下必须产生相同结果。不用随机判定（与 `rollSideEffectivePower` 的 `×[0.8,1.2]` 区分开——那是战力掷点，不是技能触发掷点）。
5. **数据驱动**：技能映射存 `GeneralSkills.ts`，不硬编码在战斗逻辑中。AI 未来给新将领分配技能时，直接编辑 `GeneralSkills.ts`。
6. **上限保护**：战力倍率最多 2 层叠加（1.44×）。战后恢复总量不超过初始兵力 80%。这些是硬编码的安全阀，不随技能数据变化。
