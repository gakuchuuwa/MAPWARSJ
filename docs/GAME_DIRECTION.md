## 直播观赏改进路线（2026-06-16，待拍板 / 部分已实装）

> **产品定位再强调**：本作是**不出声解说也能看懂**的历史大乱斗观察赛。评判标准永远是「**好不好看、有没有悬念**」，不是传统 4X 的「好不好玩」。镜头、军情字、专名、名将对撞，优先级高于美术细枝末节。

### 最重要的事（按优先级）

| 优先级 | 什么最重要 | 为什么（直播） | 现状 |
|:---:|---|---|---|
| **1** | **别一家独大** | 满屏单色 = 没悬念 = 直播死局 | ✅ 枪打出头鸟已实装（`LEADER_HUNT`） |
| **2** | **画面叫得出名字** | 无声观众靠旗号、番号、据点名认人 | ✅ 番号随城；&lt;4万 **50% 出番号**（仅精锐 / 精锐+将 两档） |
| **3** | **名将有脸、有对撞** | 「白起对廉颇」比两坨色块值钱 | ✅ 名将归势力；&lt;4万 **50% 出将**（仅将 / 精锐+将）；≥4万 必带将 |
| **4** | **镜头有焦点** | 跟拍 / 导播 / 军情 feed 当字幕 | 部分实装；导播分级、点击定位待加强 |

**铁律**：改观赏参数前，先想「无人解说时，这一眼观众能不能看懂」；改平衡参数后，跑 `npm run sim` 看终局存活势力数与「领先者破百城」比例。

### 军情播报规范 v1（2026-06-16）

> **用途**：右侧“军情”面板（`BrawlFeedPanel`）的播报口径与排版规范。  
> **目标**：无声直播也能看懂；同类事件一眼识别；句式稳定，便于观众“学会阅读”。

#### 0. 术语与红线

- **势力 ≠ 旗号**：军情文本里“谁”一律用 **势力全名**（`getFactionName()`），**不**用旗面短字（见 `player-facing-terminology.mdc`）。
- **据点**：用地图城名（`cities_v2.name`）。
- **将领**：仅当该军团确实带 `generalId` 且有档案时显示；**复国类不带将领**（规则见下）。
- **排版**：空格、标点必须稳定；不要在不同类型之间随意换句式。

#### 1. 六大类军情（游戏内真实表现）

1. **【灭】势力覆灭**（攻方占守方最后一城）

`【灭】 前202年冬 汉国 韩信 率 轻勇骑 攻占 彭城，西楚亡。`

2. **【歼】全军覆没**（攻/守任一军团覆没）

`【歼】 234年夏 蜀国 王平 率 无当飞军 于 街亭 全军覆没。`

3. **【复】势力复国**（起义复国 / 于某据点复国）

`【复】 1127年夏 大宋 于 汴梁 复国。`

> 说明：**复国类不带将领与兵种**，只报“势力 + 据点”。

4. **【征】远征**（跟拍军团誓师/功成；低频大事）

`【征】 1220年夏 蒙古 成吉思汗 率 怯薛宿卫 誓师远征 撒马尔罕。`

5. **【隘】关隘防御**（攻打 `type: pass` 据点）

`【隘】 前389年冬 魏国 吴起 率 魏之武卒 攻打 函谷关，驻军拒险而战。`

6. **【占】攻克大城枢纽**（大城/文化枢纽级占领；用于强调“关键城”）

`【占】 618年冬 唐国 李世民 率 玄甲军 攻占 中原 长安。`

#### 2. 统一模板（写代码时照此拼装）

- **带将领与精锐**（灭 / 歼 / 征 / 隘 / 占）：

`【类】 {年季} {势力} {将领} 率 {精锐/军团名} {动作} {据点}{后缀句}`

- **不带将领**（复）：

`【复】 {年季} {势力} 于 {据点} 复国。`

> 注：当军团无将领时，允许省略“将领”字段，但不得破坏整体句式节奏。

### 家城被攻 vs 第三方排队（2026-06-16 定案，**部分实装**）

> **主人澄清**：实机里军团「像回援一样折返」是**观感描述**；**游戏没有「回援」这条正式设定**（无配置、无军情、无行为树节点）。先前误把隐藏逻辑当成回援设计，本节按「**无回援**」重写。

#### 游戏里实际有什么（代码真相）

| 名称 | 是否存在 | 触发 | 玩家是否感知 |
|---|---|---|---|
| **回援** | ❌ **无** | — | — |
| **收复失地** | ✅ | 出发点 `homeCityId` **已易手**；`resolveRecaptureTarget` | 选目标行军，无专用军情 |
| **城市防御增援** | ✅（隐式） | 任意城开战瞬间 `triggerDefensiveReinforcements`；同旗 idle、非战斗、**hex ≤2** | ❌ 无文案 |
| **战场援军** | ✅ | 已开战；`BattleReinforcementPoll` 圈内编入 | 战报「援军赶到」 |
| **第三方攻城排队** | ✅ | 到别城等战场结束；`IsWaitingSiege` 压住行为树 | 原地等 |

马鲁鲁德**未失守** → 不走收复；彭迪↔马鲁鲁德 **3 hex** → 按理也不走 2 hex 防御增援。若军团仍折返，是**未文档化副作用**（多为防御增援误拉、或当时未真在排队），**不是**设计内的回援。

#### 定案：**无回援**

| 级 | 条件 | 行为 |
|:---:|---|---|
| **0** | 远征 / 剧本 | 家城失守也不回师 |
| **1** | 出发点**已易手** | **收复**出发点 |
| **2** | 第三方攻城排队 | **等到该城战场结束**；家城仅被攻、未失守**不撤队** |
| **3** | 正常进攻 AI | 近敌抽签 / 推进 |

**铁律**：① 没有「家城被攻就回家」；② 只有失守才强制改目标去收复；③ 防御增援须收口（不拉排队中的军、不拉远距野战军），避免再现幽灵折返；④ **不要**新增回援节点或 `HOME_DEFENSE_RECALL`。

#### 你的案例（定案后）

1. A 攻彭迪；C 到彭迪 → **排队不撤**。  
2. D 攻马鲁鲁德（城仍古尔）→ 仅驻军 + **≤2 hex** 空闲同旗军可被动增援；C 在彭迪应**继续等**。  
3. A 打下彭迪 → C 出队后正常 AI 选目标（收复**仅当**马鲁鲁德已丢）。  

**实装（2026-06-16）**：`triggerDefensiveReinforcements` 排除第三方排队与远征/剧本军；`IsWaitingSiege` 在出发点失守时出队走收复。

### 精锐部队战力 ×1.2（2026-06-16 拍板）

**结论：对，而且已经实装**——不是新提案。

| 项 | 说明 |
|---|---|
| **机制** | `isElite` 或远征/剧本行军锁目标 → `getCampaignLegionCombatMultiplier` → `CAMPAIGN_LEGION_MULT`（**1.2**） |
| **代码** | [`CultureCombat.ts`](../src/systems/CultureCombat.ts) + [`GameConfig.ts`](../src/config/GameConfig.ts) |
| **战报 UI（拍板）** | **不用独立技能名**（不设「锐士当锋」「精锐」等系统技）。技能条主标签 = **番号专名**（陷阵营、玄甲军、秦锐士…），副行只写 **`×1.2`** |
| **禁止文案** | **禁止**「剧本×1.2」「精锐×1.2」——观众只认番号，不认内部「剧本军团」术语 |
| **与名将技区分** | 名将 **③侵掠如火** 等 = 开局战术 ×1.2，**要名将档案**；番号 ×1.2 = 劲旅被动，**不依赖 generalId** |
| **与援军区分** | **合兵一处** = 援军 luck；**拒险而守** = 关隘守军 ×1.2 |

## 大方向（三句话）

1. **这是什么**：真实亚洲大地图上的「历史大乱斗观察赛」——秦、汉、大清、德川等百余势力 AI 自己打，玩家像上帝 / 史官在旁边看。
2. **你怎么玩**：几乎不操作。选阵容、调速暂停、跟拍某支军团或让导播带你跳热点、看事件流；偶尔触发一次史实天灾。
3. **怎么慢慢做好**：先让**战斗结果可信**（像真历史），再让**画面好看**（跟拍、播报），最后加**灾年、迁都**等涌现故事。

## 战斗系统现状（先看懂这个，再谈改）

战斗分两条路，**关隘加成只在攻城战那条路上**：

| | 野战 | 攻城战 |
|---|---|---|
| 触发 | 两支行军军团在道路上相撞（[`LegionFieldBattle.ts`](../src/legion/field/LegionFieldBattle.ts)） | 军团抵达敌城，守方是城驻军（[`SiegeManager.ts`](../src/combat/SiegeManager.ts)） |
| 战力公式 | 兵力 × 文化区系数 × 运气(0.8~1.2) | 兵力 × 文化区系数 × **关隘系数**(pass 城守军 ×1.2) × 运气 |
| 地理影响 | 通过**文化区系数**体现（按军团出身，不按脚下地皮） | 关隘城守军加成 |

军团类型只有两种，由文化决定（[`CultureFormations.ts`](../src/types/CultureFormations.ts)）：

- **纯骑军团**（三角阵）：草原、青藏、西域 3 文化
- **步骑军团**（3×3 方阵）：其余 11 文化

地形判定（`LandTerrainSystem`：平原 / 山地 / 海）只影响**行军速度**（骑兵平原快、山地慢），不进战斗公式——**这是定案，不是缺陷**（见下方否决记录）。

> 旧的"步克骑、骑克弓"兵种克制表（UnitCounterSystem）系早期遗留死代码，已于 2026-06-10 删除，**不要再引用**。

### 战斗设计定案（项目主人已裁定，AI 勿再提案）

| 提案 | 裁定 | 理由 |
|---|---|---|
| 野战按碰撞点地形加减战力（如"山地削骑兵"） | **否决**（2026-06-10） | ① 行军全部走史实道路网，碰撞发生在驿道上，不存在"山间野地遭遇"前提；② "山地骑兵弱"无史实依据——吐蕃、吐谷浑皆高原山地骑兵且战力强悍。不符合历史事实的设定一律不加 |
| 兵种三角克制（步克骑等） | **废弃**（死代码已删） | 军团只分纯骑/步骑两类，不按兵种分军 |

地理对战力的影响**已有**且足够：文化区系数按军团**出身**定强弱（草原/东北/青藏野战 ×1.2——吐蕃骑兵到哪都能打），关隘系数保护险要城池。**按出身，不按脚下地皮**，这是本项目的战斗地理原则。

### 战斗立绘三级分配（2026-06-21 定案）

> **单一真理**：AGENTS.md **§十三** + `src/config/portrait_defaults.ts`。废弃 `portraits/`、`avg/`。

## 反雪球：枪打出头鸟（2026-06-15 定案，已实装）

> **背景**：本项目是观赏/直播产品，判断标准是「**好不好看**」而非「好不好玩」。一家独大 = 满屏单色 = 没悬念 = 直播必死。这是观赏产品的头号生死问题，优先级高于一切美术/名将细节。

**无头推演实测铁证**（`npm run sim`，30 局 × 300 年，648 城）：

| 规则 | 领先者达 100 城 | 第 200 年灭国数 | 终局存活势力 |
|---|---|---|---|
| **关闭（baseline）** | 100% 的局（第 78 年） | 544 / 549 | **4** |
| 复国提频（dynasty） | 100%（第 90 年） | 527 / 549 | 22 |
| **枪打出头鸟（hunt）** | **0%（从不达到）** | 457 / 549 | **34–81** |

结论：**复国提频治标不治本**（领头羊照样破百城，只是身后城池易主多）；**唯一掐住雪球的杠杆是「枪打出头鸟」**——某势力过大，其它军团概率性掉头打它。

**机制（已实装）**：[`TargetEvaluator.pickTarget`](../src/ai/TargetEvaluator.ts) 选目标时，若存在据点数 ≥ `CITY_THRESHOLD` 的领先势力，各军团有 `PROBABILITY` 概率改打**该领先势力最近的城**。参数集中在 [`GameConfig.AI.LEADER_HUNT`](../src/config/GameConfig.ts)：

| 参数 | 当前值 | 含义 / 调参方向 |
|---|---|---|
| `CITY_THRESHOLD` | 40 | 领先势力据点达此数才被合击。**越低 = 越早压制** |
| `PROBABILITY` | 0.20 | 各军团掉头打领先势力的概率。**越高 = 雪球压得越死**（主杠杆） |

**为什么是 th40 / p0.20**：实测此组合既打掉「第 78 年一家独大」的灾难（终局存活 34 势力），又保留 **37% 的局能在约第 276 年苦战分出胜负**——既反雪球、又留统一的叙事高潮。纯 p0.30 会变成永不结束的混战，反而失去高潮。**AI 调参先跑 `npm run sim` 看「终局存活势力数」和「领先者达 100 城的比例」两个指标，别凭感觉改。**

> 这是「远征目标优先打边缘强国」的系统化、不点名版本——不写死打日本/朝鲜，谁雪球打谁，涌现而非脚本。

## 名将归势力（2026-06-15 定案，已实装）

> **背景**：观赏产品最该有的是「白起对廉颇」这种历史名场面。早先「名将只给远征军/跟拍军团」是错的——那永远是玩家名将打 AI 无名，单边碾压，且 AI 永不出名将。

**定案**：名将是**势力的属性**，**AI 也有**；&lt;4万 可单独出将（「{城名}军团」+脸），也可与番号同出。

**军团分层（2026-06-16 四档摇奖，已实装）**：

| 层 | 条件 | 拥有 |
|---|---|---|
| 普通据点军团 | 无番号映射，或 &lt;4万 摇中「普通」档 | 「{城名}军团」 |
| **仅精锐** | 有番号、&lt;4万 摇中「仅精锐」 | 番号 + ×1.2；无名将 |
| **仅将领** | 有番号据点、&lt;4万 摇中「仅将领」 | 「{城名}军团」+ **名将脸**；**无** ×1.2 |
| **精锐 + 名将** | &lt;4万 摇中「精锐+将」，或 **≥4万**（必） | 番号 + 脸 + ×1.2 |
| **远征军** | 当前跟随的那支精锐（同时仅一支） | 同上 + 远征令 +「因粮于敌」 |

- 配置：`GameConfig.LEGION_TIER` 四档概率；番号 [`ExpeditionLegions.ts`](../src/data/ExpeditionLegions.ts)；将领 [`FactionGenerals.ts`](../src/data/FactionGenerals.ts)。
- 分层：[`LegionManager.applyLegionTier`](../src/legion/LegionManager.ts) + `tickLegionTiers`（长到 4万 补精锐+将）。出生定档，精锐/将领**不降级**。
- **精锐战力加成**：`isElite` 享 `CAMPAIGN_LEGION_MULT`，但**不吃「断粮不回师」**（那只跟远征令 `expeditionTargetCityId` 走）。
- **生效**：[`GeneralSkillCombat.canUnitUseGeneralSkills`](../src/combat/GeneralSkillCombat.ts) 只看「带 generalId 且有档案」，**不要求跟随/远征** → AI 名将同样触发；开局战术/战略**攻守双方各自结算**。

**AI 调参注意**：现行平衡推演（`npm run sim`）**不含**精锐/武将技层，精锐铺开后要另行验平衡。

## 一据点一将一精 + 旗号随占城（2026-06-22 主人定）

### 数据链

### 代码

**禁止**：`getFactionGeneral(city.factionId)` 决定守城/募兵将；代码或文档写死固定人名配对。

## 2026-06-10 优化评审定案（v1.4）

> 本节为项目主人 2026-06-10 评审后的拍板记录，日后 AI 不得与之相悖；标注「待拍板」的细节，开工前先问主人。

### 已拍板

| 决定 | 说明 |
|---|---|
| **跟拍补兵 = 御驾亲征** | `FollowResupplySystem`「仅跟拍军团可补兵」**不是 bug，是玩法**：观察者也有自己想赢的队伍。定性为玩家唯一的战力干预，已写入白名单；UI 须给跟拍军团加「亲征」标记，让玩家知道自己正在影响战局。**禁止**任何 AI 以「公平性」为由把补兵改成对全军团生效 |

| **批量推演开工（2026-06-11）** | 主人判断：不存在跟拍军团时，所有势力行为同质 → 推演无观察者偏差，统计有效。实现：`scripts/sim/`（无头模拟，复用真实数据 cities_v2/factions/StartingCapitals 与真实战斗数学 GameConfig/CultureCombat/CityConfig，运行 `npm run sim`）。简化项（道路网→直线行军等）在脚本头注释中逐条声明 |

### 剧本军团（2026-06-12 主人拍板，秦已实装实测通过）

| 出生 | 开局即在出生城生成（`spawnAtStart`）；挂名将立绘（`generalId`）、精锐番号（不叫泛称「秦军」，用「秦锐士」） |

#### 远征精锐番号（2026-06-11，数据规范见 AGENTS.md §十二）

跟拍军团解锁远征后，军团名改为 **远征番号**（3–6 字史料专名）。玩家此时同时看见 **三样** 东西，必须是 **三个不同专名**：

| 旗号（旗面） | 据点名（地图） | 远征番号（军团名） |
|-------------|---------------|-------------------|
| 例 `山越` | 例 `宛陵城` | 例 `丹阳兵` |

**安置（不可颠倒）**：① 定据点 → ② 该点定旗（§4.1）→ ③ 该点定番号 → 写入占该点的 `factionId`。**势力、精锐、时代彼此无绑定，都只和据点有关**（AGENTS.md §12.2）。精锐不另占城；乡土兵回故里（白杆兵@石柱）。

**旗号禁字（2026-06-16）**：旗面 1–2 字 **不得含「州」「国」**（例：势力名 **宿州** → 旗号 **`宿`**，非 `宿州`；**徐国** → **`徐`**）。细则见 AGENTS.md §4.0「旗号禁字红线」。

**三者防重（主人多次强调，详见 AGENTS.md §12.1.1）**：

1. 旗号、据点名、番号 **两两严格不相等**
2. **默认**不得共享汉字（旗号∩据点、据点∩番号 **始终**禁止）
3. **旗号∩番号**：**仅** §4.1 **民族 / 政权 / 家族** 可重复（如 `魏`+`魏武卒`）；军镇专名作旗号时须与番号错开（`魏博`+`天雄军` ✅，非 `天雄`+`天雄军`）
4. 势力 **禁止用人名**；写入后跑 `npm run expedition:triple-check`

**首轮推演发现（v0，2026-06-11，100 局 × 300 年，详见 `scripts/sim/sim-report.json`）**：

- **游牧没有雪球**：现行系数下草原 0 胜、青藏 3%、东北 8%——「就地取粮+速度+野战1.2 会不会太强」的担忧，在战略层暂未成立（断粮上线后须重跑验证）
- **真正失衡的是边角文化**：日本文化胜率 **65%**（edo 32 胜 + riben 30 胜）、中亚 12%、朝鲜 6%；而中原仅 2%——边角少面受敌 + 大城驻军无人打扰养到爆仓出 9 万大军；中原是四战之地。注意 v0 直线行军无海洋阻隔，日本优势可能被低估也可能被高估，待 v1 接入道路网验证
- **雪球与灭国节奏**：领先者平均第 27 年达 30 城、第 58 年达 100 城；第 50 年平均已灭 420/495 国；终局平均仅存 3 个势力。复国每局约 175 次，能延缓但挡不住头部
- v0 简化（直线行军、无援军圈、关隘无咽喉效应）写在 `scripts/sim/headless.ts` 头注释，解读数据先看那里
- **A/B 实验（2026-06-11，同种子各 100 局）**：日本文化胜率——基线 65% / 五级攻防 43% / 断粮 59% / 五级+断粮 52%。四份报告 `scripts/sim/sim-report-{baseline,tiers5,supply,tiers5+supply}.json`。结论：两个机制方向都对、可叠加；日本残余优势是边角地理，下一个候选杠杆是**海路行军/跨海作战惩罚**（元日战争史实可考）

**军团 AI 两种模式（术语定案）**：
- **基础模式**：现行沙盒 AI——「道路最近 N 座敌城」均匀抽签（`GameConfig.AI.TARGET_NEAR_POOL = 3`，即「随机三据点」）
- **远征模式**：一条线打向指定文化中心，中途照常接战 / 攻城，不改目标

### ~~断粮模式~~（2026-06-11 实装，**2026-06-12 主人裁定整体删除**，由「分级战后恢复」替代，见下节）

**废止原因**：「家城产出优先补自家军团」关闭了大军团的唯一生成通道——"一城一军保护伞下家城囤积→军团死后爆发出大军"。它让军团家城城防长期停在 ~2000、全图大军团绝迹（御驾亲征抽兵也跟着变弱），主人实测"很难有 5 万军团"后查明此因果，裁定删除。曾有的推演数据（日本 52%、断粮 264 次/局等）仅作历史记录，勿再引用。

**保留的部分**（非断粮机制、是游戏原生行为）：
- **家城失守 → 强制回师收复**（行为树 `resolveRecaptureTarget`，所有文化无豁免）
- **远征军豁免**：远征模式军团不回师，直至占领目标或全军覆没（2026-06-11 拍板，仍有效）
- 观赏价值不损失：「失家回师」每局仍 ~218 次（新规则推演实测）

### 分级战后恢复（2026-06-12 主人设计，已实装 + 推演验证）

**核心：军团兵力的恢复不靠后方输血，靠打胜仗——打得越大，恢复越多。** 替代旧的一刀切 30% 伤兵恢复（`WOUNDED_RECOVERY_RATE`，已删）。

| 战斗场景（胜方） | 恢复本场战损的比例 |
|---|---|
| 打下关隘 | **10%** |
| 打下小城 | **20%** |
| 打下中城 | **30%** |
| 打下大城 | **40%** |
| 野战获胜 | **50%** |

- 守城方胜利同样按本城等级恢复（伤兵就在城中）
- 实现：`GameConfig.COMBAT.POST_BATTLE_RECOVERY` + `CombatSystem.getPostBattleRecoveryRate`（1v1 与多方区域战 `BattleField` 两条结算路径都已接入）；推演程序读同一份配置表
- 据点产出（大400/中300/小200/关100）恢复为**全进城防**——大城囤积爆发式出兵通道重新打开
- **推演验证（50 局 × 300 年，五级攻防 + 本规则）**：日本胜率 **40%**（断粮版 52%、最初 65%），东北 22%、中亚 14%、青藏 10%——目前最健康的分布；失家回师 218 次/局，戏剧密度保持；军团峰值平均 3.8 万、5 万+军团 每局 0.2 支（大军团回到"后期长寿军团的家城囤积爆发"模式，稀有但存在）

### 遏制边缘势力（2026-06-11 实验与评估）

推演证实**边缘地利**（日本/朝鲜/中亚/吴哥：少面受敌 + 大城安心爆兵出 9 万大军）是头号失衡源，复国系统挡不住。主人提出三个方案，评估如下：

| 方案 | 评估 | 结论 |
|---|---|---|
| 减少边缘据点 | ❌ 不推荐：据点皆史实可考，为平衡删点违反「史料优先」（AGENTS §一）；且边缘强不在据点多，在没邻居 | 否 |
| 远征/进攻目标优先日本朝鲜 | ✅ 方向对，但点名硬编码不自适应 → 系统化为**枪打出头鸟（合击）**：领先势力 ≥ N 城后，各军团以概率 p 改打领先者最近的城。即 roadmap 阶段三「一家过大则遭合击」的机制化，谁雪球打谁、不冤枉人 | 已做进推演（`--hunt --huntp --huntth`） |
| 库页岛—北海道路线 | ✅ 推荐进真游戏：给日本开第二战线，直接改地理拓扑。史实可考：元代征骨嵬（库页岛白主土城为元代遗址）、松前/箱馆皆 1900 前可考地名。需新增据点 + 道路（FactionEditor / VectorRoadEditor 流程，AGENTS §2 间距与可考性照守） | 待主人发起数据层录入 |

**合击三档实测**（在 tiers5+supply 之上，100 局同种子）：

| 档位 | 参数 | 日本胜率 | 开张文化数 | 胜者平均城数 | 终局存活势力 |
|---|---|---|---|---|---|
| 无合击 | — | 46% | 9/14 | ~550 | 4 |
| 强 | p=0.3, th=30 | **20%** | **14/14** | — | 79（永无统一） |
| 中 | p=0.25, th=50 | 38% | 11/14 | 82 | 45 |
| 弱 | p=0.15, th=80 | 64%（无效——出手太晚） | 7/14 | 405 | 8 |

**结构性发现：合击强度 = 悬念 ↔ 大一统结局的硬权衡。** 全图军团都扑向领先者，等于把帝国规模封顶在阈值附近；合击时机（th）比力度（p）更关键，80 城才出手已经拦不住了。

**主人裁定（2026-06-11）：合击封存，暂不实装。** 等日后做**外交系统**时一并设计——围攻领先者本质是外交行为（合纵结盟），届时做成正式的结盟/宣战机制，而不是现在的概率硬编码。上表三档实测数据保留，作为外交系统设计时的力度参考。推演程序的 `--hunt` 系列旗标保留可用。

### 王朝周期律（2026-06-12 主人提案，推演已验证，**档位待拍板**）

**主人论点：王朝最大的敌人是不公——土地兼并导致农民起义。加强复国系统，而不是天灾**（灾年方案同日被否：天灾是外生随机数，民变是内生因果）。

三条规则（零新数值条，全在 RebellionSystem 上加强；推演 `--dynasty --dyscale N --dyrecruit M`）：

| 规则 | 机制 |
|---|---|
| 兼并越广民变越频 | 每年额外民变轮数 = floor(最大势力城数 / dyscale) |
| 天高皇帝远 | 民变在最大势力的**占领城**中点燃，距其龙兴之地（开局城）最远的前一半优先——陈胜吴广起于大泽乡 |
| 从者云集 | 起义军 = max(开局驻军, 现驻军½) + floor(帝国城数/10)×dyrecruit 流民 |
| 灭国文化优先 | 照旧（与远征同化联动） |

**三档实测（50 局 × 300 年，五级攻防 + 新战后恢复之上，同种子）**：

| 档位 | 民变/局 | 日本胜率 | 开张文化 | 胜者平均城数 | 达 100 城年份 | 终局存活 | 军团峰值 |
|---|---|---|---|---|---|---|---|
| 无周期律（现行） | 0 | 40% | 9/14 | 589（大一统） | 62 年 | 3.4 | 3.8 万 |
| 温和 dyscale=60 | 666 | 26% | 6/14 | 266（无统一） | 77 年 | 20 | 2.6 万 |
| 强 dyscale=30 | 989 | 20% | **12/14** | 162（无统一） | 96 年 | 32 | 2.3 万 |

**结构性发现（与合击实验同型）：周期律强度 = 悬念/多元 ↔ 大一统的硬权衡。** 任何档位下 AI 均无法统一（强档帝国被封顶在 ~160 城）；同时民变频繁打断大城囤积，军团峰值跌至 2.3-2.6 万，**5 万远征解锁在周期律下几乎绝缘**——若采纳须连同远征门槛/名将精锐（质量补数量）一并考虑。

**主人定调（2026-06-12）**：不统一没关系——本作主角是**军团/精锐部队完成历史使命**（秦灭六国、垓下合围、蒙古西征、明北伐），不是势力；但也不能极端到永无大一统。

**渐进式天命（--tianming，50 局实测，推荐档）**：民变轮数 ×(1 − 持有文化中心数/15)；握满 15 座 → 民变止息（天命所归），丢中心即恢复。攒法统的过程本身就在平民心，且与远征系统天然咬合——远征目标恰是文化中心，**打中心 = 既扩张又靖民**，玩家军团的使命直接为王朝续命。

| 配置（均含五级攻防+新战后恢复） | 大一统(>500城) | 胜者城数(均/最大) | 民变/局 | 达100城 | 终局存活 | 军团峰值 |
|---|---|---|---|---|---|---|
| 现行（无周期律） | 50/50 必然 | 589 / — | 0 | 62 年 | 3.4 | 3.8 万 |
| 温和周期律（每60城1轮） | 0/50 绝无 | 266 / 323 | 666 | 77 年 | 20 | 2.6 万 |
| **温和+渐进天命（推荐）** | **7/50 偶有** | 363 / 631 | 417 | 74 年 | 10.6 | **3.5 万** |

推荐档恰合主人定调：大一统从必然变成 14% 的"伟业"；多极乱世为常态（终局平均存活 10.6 势力）；军团峰值基本保住（3.5 万，远征 5 万门槛仍可及）；后期垃圾时间被每年数轮民变填充。残留问题：日本胜率回升至 44%（天命衰减让最强者受益），可在实装后用中心持有判定细则再调。

**注意：以上全部只改了推演脚本（`scripts/sim/headless.ts` 的 `--dynasty --dyscale --dyrecruit --tianming` 旗标），真实游戏的 RebellionSystem 尚未动**——主人拍板后再实装。

### 名将与战斗加成（2026-06-12 策划评审）

**背景**：战斗是「数字碰数字」的闭式结算（开战定胜负），观赏性薄。借鉴 CK3 / 信长 / 三国志的共同内核——**有名有姓、能让观众产生感情的角色**（文明的内政/建造内核不适用，本作已砍生产）。无解说直播尤其需要一个「主角」可揪心。已删死配置 `MORALE`/`FLANKING`（点线大战略图无战术界面，士气侧翼仍是看数字掉，徒增 BUG 面）。

**战斗单位术语（主人 2026-06-12 定，全项目统一用词）**：**军团**（出征的野战军）、**援军**（开战后才赶到编入的同旗军团/驻军，曾称"生力军/新锐"已弃用）、**精锐**（虎豹骑/玄甲军等名将招牌部队）、**驻军**（据点城防）。

| 项 | 评审结论 | 做法 |
|---|---|---|
| **文化系数 固定 vs 随机** | ✅ **保持固定**（已决） | 身份要可学（蒙古野战猛是定数）；随机已由 luck[0.8,1.2] 承担，且在对的位置——兵力接近才爆冷，差距大强者必胜。系数再随机=双重噪音，毁掉已调平的格局 |
| **援军加成** | ✅ **已实装**（主人 2026-06-12 拍板 ×1.2 固定，不推演） | 开战后才编入的援军（`BattleField` waveIndex≥1）战力 **×1.2** 计入强弱重算（`GameConfig.COMBAT.REINFORCEMENT_MULT` + `adjustedPowerWithReinforcement`）；开局已在场的主力（waveIndex=0）无加成。养精蓄锐冲击鏖战之敌（围魏救赵/垓下），让「韩信大迂回」从多 N 兵变成多 N 个更能打的兵 |
| **战斗面板显示援军** | ✅ 采纳（纯 UI） | `CombatUI` 列出主力 + 各路援军名（现在只显示 `+1`）；过多时显示前几个 + 「余 X 部」。「援军赶到！」是直播紧张感来源 |
| **远征加成 + 远征武将加成** | 🔀 **建议合并为「名将系统」** | 远征军的强**来自它带的名将本身**，不是独立的「远征 ×20%」魔法。名将 = 名字 + **一个**招牌特长（野战/攻城/突击 → 一档系数）。虎豹骑等精锐 = 名将的皮肤+特长，不另起系统。待主人最终拍板 |
| **武将技翻盘** | ✅ 采纳（最高价值，主人 2026-06-12 定型为「危机触发事件」） | **不是常驻乘数，是危机时刻一次性事件**——败方兵力跌破阈值时，带名将一方 P% 概率触发，且**触发了也可能输**（势均力敌才有戏，非翻盘必胜键）。三种效果各名将固定带一种：①**回血**（注入一笔兵）②**暴击**（瞬削胜方一笔兵）③**临时不掉血**（暂停掉血数秒）。①②复用引擎现成「援军编入后强弱重算」（`BattleField.refreshPredictedSidesFromTotals`），近零 BUG。③最佳——为奔袭援军争取抵达时间，「武将技顶住 vs 援军赶到」双倒计时 = 戏剧顶点 |

**✅ 叠乘风险已由武将技「危机事件」设计化解**：武将技**不进战力计算**（顺风时永不触发，只在危机掷一次），故不与赢面乘数相乘。仍需盯的乘数链只剩 **文化 × luck × 援军(1.2) × 名将特长**（后两项为新增；援军 1.2 已定，名将特长待定）。铁律：**名将特长 / 武将技触发率先推演 100 局再定**，绝不拍脑袋上线。

**名将系统与「编年史」联动** = 本作版的「CK3：角色 + 涌现故事」：名将攒战绩（打下哪些城、连胜几场）、战死即噩耗，一局结束的战后编年史里有名有姓——观众跟的是「卫青的军团」，不是「长安军团」。

|---|---|

### 设计联动（为什么这几件事是一个闭环）

**远征（目标）+ 战损不可逆（风险：兵力只能靠胜仗按比例回血）+ 御驾亲征（玩家维持兵力的手段）= 观赏核心。**（家城补给与就地取粮均已废止，文化差异由五级攻防承担）
没有断粮，远征只是一段结局已知的长途行军；有了断粮，农耕大军团远征日本是「九死一生的豪赌」，游牧军团西征是「史实重演」——观众盯着的就是这条补给线。

### 工程待办（小，不影响玩法）

- `getEuclideanDistance` 直接用经纬度度数：经度方向实际公里数随 cos(纬度) 缩小，漠北（北纬 48°）的接战圈 / 补兵半径实际比江南窄约 30%。如需全图一致，lng 差乘 cos(lat) 校正即可
- 战力结算等纯函数补最小单测（`npm test` 目前是占位符），作为批量推演的地基
